# 月薪喵来源角色一致性合成

## 结论

`yuexinmiao1` 的公开宠物包只提供 `pet.json` 和 `spritesheet.webp`，没有 SVG 路径、角色源代码或可直接修改的动作工程。手工猜测角色轮廓会造成头身比例、线条和表情与来源角色明显不同，因此不再使用手绘通用猫骨架。

## 当前实现

- 每个 CCM 业务动作仍是独立 SVG 文件。
- SVG 内部嵌入来源 WebP 图集，并按 192×208 单帧边界选择角色帧，因此角色外观与来源帧完全一致。
- 清单、电脑、放大镜、扫帚、资料箱、铃铛、完成勾和其他业务道具由 CCM 独立绘制。
- 不同动作选择来源图集中的不同手势或姿态，并通过 SVG CSS 增加对应运动。
- 音乐动作只使用身体横移、轻跳和摆动，不显示耳机或音符。
- 原图集层的动画变换基于单帧 viewBox，而不是整张 1536×1872 图集，避免横向运动时角色被裁掉一半。

## 资源与来源

- 来源作者：`kiffin`
- 来源页面：<https://codex-pet.org/zh/pets/yuexinmiao1/>

- `SOURCE.txt` 已说明业务 SVG 使用来源角色帧，并独立绘制业务场景和动画。
