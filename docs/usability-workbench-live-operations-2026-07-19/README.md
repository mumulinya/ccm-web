# 我的工作台实时与操作能力升级

## 范围

本轮在已完成的指挥中心布局上补齐六项实际能力：实时更新、具体执行进度、待处理分类、资源快捷操作、个人布局和离线恢复。没有新增平行任务系统，继续使用现有任务、项目、群聊和定时任务接口。

## 实现

### 实时状态

- 新增 `/api/usability/workbench/stream` SSE 数据流，连接后立即发送完整快照。
- 服务端每 2 秒检查一次状态签名，仅在数据变化时推送更新；每 10 秒发送心跳。
- 前端移除旧的 15 秒轮询，EventSource 断线后自动恢复，并进行一次安全 HTTP 拉取。

### 任务进度与待处理分类

- 工作台任务增加当前步骤、可信进度百分比、开始时间、运行时长、最近动作和等待原因。
- 没有真实百分比时显示不确定进度动画，不伪造完成比例。
- 待处理事项分为“需要确认、执行失败、等待补充”，顶部同步展示总数量。

### 资源快捷操作

- 项目可从工作台直接启动或停止；停止前必须二次确认。
- 群聊可直接打开到精确群聊。
- 定时任务可直接启用或停用。
- 操作完成后立即拉取状态，后续变化继续由 SSE 推送。

### 个人布局

- 用户可以隐藏目标区、快捷入口、待处理、执行中、近期交付或资源栏。
- 快捷入口支持上移和下移，布局保存在当前浏览器的 localStorage。
- 支持一键恢复默认布局。

### 离线与过期提示

- 最近一次成功快照保存在 localStorage，首次加载失败时仍展示可用数据。
- SSE 中断或心跳超时后显示明确的缓存时间、错误原因和“立即重试”操作。
- 新数据到达后自动清除过期状态，不需要刷新页面。

## 安全边界

- 浏览器验证只检查按钮和状态，没有实际启动或停止项目、群聊或定时任务。
- 原任务、会话、项目和定时任务数据未清理、迁移或伪造。
- 本轮没有调用任何付费 Provider。
