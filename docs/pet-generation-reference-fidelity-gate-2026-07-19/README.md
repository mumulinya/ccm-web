# 宠物参考图身份保真门禁

## 问题

旧流程会把原参考图直接交给全局视觉模型生成 SVG，但首版只校验 SVG 安全、画布、元素数量和动画结构。模型即使正确提取了发型、配件和服装，也可能在绘制时退化成通用 Q 版人物；这种结果仍会进入用户确认。

本次示例中，任务的 `designBrief` 已正确识别长直发、左侧发夹、奶油白吊带、蕾丝外搭和层叠裙装，但候选 SVG 擅自放大头部、缩短身体并简化成通用裙装。根因是缺少“原图与候选渲染图”的首版身份复审，而不是上传图片丢失。

## 新流程

1. 全局视觉模型读取原参考图并锁定比例、脸发、服装拓扑、姿态气质、左右配件和禁止偏移项。
2. 模型生成完整 `idle.svg` 候选。
3. 服务端将候选 SVG 真实渲染为 PNG。
4. 同一个全局视觉模型同时查看原参考图和候选渲染图，分别评分：
   - 轮廓与身体比例
   - 脸型与发型结构
   - 服装拓扑与层次
   - 配件与左右不对称
   - 用户风格意图
5. 总分必须至少 `0.80`，关键分项必须至少 `0.68`，且不得检测到通用 Q 版模板。
6. 未通过时把具体偏差作为下一候选的硬性修改要求，最多自动重做两轮。
7. 三次候选仍未通过则失败关闭，不展示为可采用的正式预览。

## Skill 约束

- `auto` 不再代表 Q 版、幼态、大头或玩偶化。
- 照片转动漫默认采用保留视觉成熟度和身体比例的半写实动漫转译。
- 不得将肩带、腰部开口、披挂外套、分层裙摆等结构改成通用连衣裙。
- 不得无依据放大头部、缩短躯干或四肢、加宽脸颊或改变人物姿态气质。
- 小尺寸允许简化蕾丝纹样，但不能改变服装拓扑和身份特征。

## 用户界面

生成记录会展示“参考相似度”、通过状态、模型结论和最多三条具体偏差。只有通过首版门禁的 SVG 才会显示“采用并生成动作”。
