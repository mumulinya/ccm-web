# 编辑项目弹窗重构

## 问题

旧 `ProjectFormModal` 只定义了少量局部样式，其余布局依赖项目父组件的 scoped CSS。子组件实际渲染时无法稳定继承这些规则，导致标签、输入框、下拉框和按钮退化为浏览器默认布局并互相挤压。

## 本次调整

- 弹窗改为自包含组件，不再依赖父页面的 scoped 表单样式。
- 桌面端使用稳定的 `620px` 最大宽度，标题区、表单区和操作区明确分层。
- 代码目录占据完整一行，并使用独立的目录浏览按钮。
- Agent 与通知平台在桌面端并排、移动端单列显示。
- 操作按钮统一使用 Lucide 图标和现有主题变量。
- 支持点击遮罩关闭和 `Escape` 关闭。
- 创建项目缺少名称或目录、编辑项目缺少目录时禁止提交。
- 旧数据包含无效通知平台时明确显示“请选择通知平台”，完成选择前禁止保存，不再展示空白下拉框或擅自猜测平台。
- 移动端改为底部面板，宽度不超过视口，内容超高时仅表单主体滚动。

## 验证

- Frontend production build 通过。
- 实际项目 `cc-connect-test` 桌面端弹窗宽度为 `620px`，页面无横向溢出。
- `390 x 844` 移动端弹窗为单列底部面板，文档 `scrollWidth` 与视口一致。
- `project-management-render-regression.mjs` 增加桌面和移动端编辑弹窗回归断言与截图。
