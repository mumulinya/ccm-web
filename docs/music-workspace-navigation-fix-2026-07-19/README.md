# 音乐工作区顶部导航修复

> 后续简化：该组顶部导航因与曲库标签重复，已在“音乐页顶部与弹幕简化”中移除。本文仅保留历史问题记录。

## 问题

沉浸式音乐页第一版把“正在播放、本地曲库、我的歌单、音乐助手、播放队列”实现成了视觉导航。其中部分入口使用普通页面锚点，激活状态固定在“正在播放”。CCM 音乐页会阻止页面级滚动，因此点击后可能不移动、移动到错误位置，且高亮不会反映当前区域。

## 修复

- 五个入口全部改为真实按钮，不再修改 URL hash。
- 导航只控制音乐页内部的滚动容器，不触发 CCM 外层页面滚动。
- “本地曲库”切换为全部歌曲视图并定位曲库。
- “播放队列”切换为队列视图并定位曲库。
- “我的歌单”打开现有歌单管理弹窗。
- “音乐助手”定位助手区域并聚焦输入框。
- “正在播放”回到封面与歌词舞台顶部。
- 顶部激活状态与曲库、队列和具体歌单视图同步；关闭歌单弹窗后恢复当前区域高亮。

## 验证

`music-render-regression.mjs` 会依次点击五个入口，验证曲库和队列切换、歌单弹窗、助手输入框聚焦、播放区返回以及 URL hash 不发生变化。随后继续执行收藏、歌单创建、添加歌曲、排序、重命名、助手搜索、重试和下载中心的桌面/移动端回归。
