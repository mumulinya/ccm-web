# 任务派发控制台改造

## 问题

- 页面缺少明确标题，任务统计以散落文本呈现。
- 所有数量为零时仍展示零值筛选器和大面积空框。
- 空状态没有提供创建任务或进入需求池的直接路径。
- 桌面端信息重心偏上，移动端状态与操作层级不清晰。

## 完成内容

- 顶部改为“派发、跟进与验收”任务控制台，补充用途说明。
- 将总任务、等待处理、正在执行、已经交付和执行失败做成稳定的五项状态带。
- 需求池、新建任务和三个视图标签统一使用 Lucide 图标。
- 概览主体改为无框内容区，减少页面区块套卡片的感觉。
- 零任务时隐藏无意义的筛选器，显示任务启动面板。
- 启动面板提供“新建任务”和“打开需求池”操作，并展示整理目标、分派执行、验证交付三步流程。
- 有任务时继续使用原执行计划、Agent 进展、验收证据、阻塞和任务操作，不改变既有业务行为。
- 移动端状态概览改为两列，任务流程改为单列，390px 下没有横向溢出。

## 验证

- `npm run build:frontend`：通过。
- `node scripts/task-dispatch-console-redesign-selftest.mjs`：通过。
- 桌面端零任务状态浏览器检查：通过。
- 390px 移动端概览和全部任务视图：通过。
- 浏览器控制台没有任务页面错误。
- 没有创建、删除或修改真实任务。
- 付费模型调用：`0`。
