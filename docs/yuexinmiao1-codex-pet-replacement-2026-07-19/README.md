# 月薪喵 Codex Pet 替换

## 目标

将 CCM 旧版简化月薪喵替换为用户指定的 `yuexinmiao1` Codex Pet，同时保留 CCM 的 20 个业务状态和桌面宠物交互。

## 资源与行为

- 原始 8×9 WebP 动画图集来自 `https://codex-pet.org/zh/pets/yuexinmiao1/`，页面标注作者为 `kiffin`。
- 待机、左右移动、挥手、跳跃、失败、等待、运行和审阅使用来源图集的原始动画行。
- 思考、规划、执行、调试、音乐、整理、搬运、提醒以及睡眠过渡等 13 个 CCM 扩展状态使用独立 SVG 动画。
- 补充 SVG 不重新画一个不同外观的猫，而是围绕来源图集帧进行不同节奏、位移和姿态编排，保证身份与画风一致。
- 补充 SVG 内嵌所需来源图集，避免浏览器或 Electron 以图片方式加载 SVG 时拦截外部子资源而出现空白。
- 网页宠物空间和 Electron 桌宠使用相同的图集行数、状态映射和补充资源。
- Web 源资源保存在 `frontend/public/pets`，生产构建会稳定复制到运行目录，避免 Vite 清理输出目录后丢失动画。
- 旧 `yuexinmiao-*.svg` 暂时保留作回滚，但默认渲染、预览和桌宠状态链路不再引用。

## 授权边界

来源包没有附带明确许可证。当前按用户确认，仅用于本地 CCM，并在两个运行资源目录记录作者、来源页面和导入日期；在获得许可前不作为可再分发宠物包发布。

## 验证

- `node scripts/yuexinmiao1-replacement-selftest.mjs`
- `npm run build:frontend`
- `npm run check`
- 浏览器逐个点击 20 个状态，确认来源图集与补充 SVG 均正常显示。
- 桌面宠物抽查待机、规划、工作、错误和睡眠状态。
- 付费模型调用：`0`。
