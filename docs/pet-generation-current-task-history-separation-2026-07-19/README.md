# 宠物生成当前任务与旧记录混淆修复

## 问题

用户选择一张新的参考图后，右侧仍直接展示旧版 Codex 图集的“已完成”记录。当前参考图尚未提交，但页面没有说明这一点，导致旧任务看起来像当前图片的生成状态。

同时，创建任务失败只显示短暂 toast，提示消失后页面没有可见错误，容易被理解为按钮没有响应。

## 修复

- 当前全局 Agent SVG 任务与旧版 Codex 图集记录分开展示。
- 旧版记录默认折叠，并明确标记“与当前参考图无关”。
- 选择文件但尚未提交时，右侧显示“参考图已选择”，说明点击按钮后才会创建任务。
- 没有选择图片或没有填写名称时，生成按钮禁用。
- 创建任务期间按钮显示“正在创建任务”。
- 创建失败会在按钮下方持续展示具体错误，同时保留 toast。
- 旧数据没有删除，历史兼容性保持不变。

## 验证

- 修复后 DOM 与截图确认当前草稿、SVG 任务和旧版记录三者已分离。
