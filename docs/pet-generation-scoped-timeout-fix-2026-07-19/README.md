# 宠物生成调用级超时修复

## 现象

全局 Agent 普通对话可以正常使用，但宠物 SVG 生成连续显示 `This operation was aborted`。最新失败任务从创建到失败为 `120.035` 秒，且 `model`、`provider`、`previewAttemptCount` 均未写入，说明请求在第一版候选返回前被客户端终止。

## 根因

共享 LLM 客户端原先只采用 `config.timeoutMs`，会覆盖调用方声明的 `defaultTimeoutMs`。当前全局配置为 `120000ms`，因此宠物生成虽然声明了 `180000ms`，实际仍在 120 秒触发 `AbortController.abort()`。

普通对话请求较短，可以在两分钟内完成；宠物首版请求同时携带参考图、完整 SVG Skill、结构化设计要求和最多 12K 输出，更容易越过该边界。模型能正常对话与宠物长请求被本地超时取消并不矛盾。

## 修复

- 共享 LLM 调用新增可选的调用级 `timeoutMs`，优先级为：精确调用覆盖 > 全局配置 > 调用默认值。
- 普通全局、群聊和项目调用没有传精确覆盖，继续遵循用户原有的全局超时。
- 宠物首版和动作批次使用 600 秒精确超时。
- 首版身份复审和动作身份复审使用 300 秒精确超时。
- 宠物请求若真正达到精确时限，会返回明确的“超过 N 秒”错误，不再只显示底层 AbortError。
- 不增加自动付费重试；用户点击重试后才发起新的模型请求。

## 验证

- 精确调用 `600000ms` 能覆盖全局 `120000ms`。
- 未传精确覆盖的普通调用仍解析为全局 `120000ms`。
- 后端与前端 production build：通过。
- SVG 宠物 mock self-test：通过，真实 Provider 调用为 `0`。
- 服务在线自检：通过。

现有失败记录保留，可由用户删除或在新版本加载后主动重试。
