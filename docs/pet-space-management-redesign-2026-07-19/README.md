# 宠物空间管理中心改造

## 目标

将宠物空间从装饰性功能集合页改造成真实 Agent 宠物的管理、预览与资源检查中心。图片生成宠物功能继续保持关闭，不恢复旧生成入口。

## 完成内容

- 删除与真实皮肤无关的 CSS 假宠物、轨道动画和深色渐变展示区。
- 首屏直接展示全局 Agent 与音乐 Agent 的真实皮肤、实时状态、详情和显示开关。
- 页面使用全宽布局，主体分为宠物库和统一详情区。
- 宠物库支持名称搜索，以及全部、内置、自定义、已使用四类筛选。
- 点击皮肤只进入预览，不会立即修改 Agent；必须点击“应用到当前 Agent”才保存。
- 详情区支持 20 个状态动画切换，并集中展示来源、格式、状态数量和应用状态。
- 集中提供应用、显示/隐藏、导出配置和删除自定义皮肤操作。
- Agent 显示名称和自定义皮肤名称继续独立管理。
- 状态策略和动作资源保留，动作资源缺失时显示明确标记。
- 配置加载和保存错误不再静默，页面提供加载状态、错误原因和重试按钮。
- 桌面宠物引擎启动、关闭使用统一提示，不再使用浏览器 alert。
- 移动端宠物详情改为底部抽屉，并避让底部导航栏。

## 数据与安全

- 没有恢复上传图片生成宠物或历史生成记录入口。
- 导出功能只下载当前宠物选择、配置和资源映射 JSON，不包含隐私数据或二进制资源。
- 浏览器验证只进行了搜索、预览状态切换、详情标签切换和抽屉开关。
- 没有启动桌宠、应用皮肤、修改名称或删除宠物。

## 验证

- `npm run build:frontend`：通过。
- `npm run check`：通过。
- `node scripts/pet-space-management-redesign-selftest.mjs`：通过。
- 桌面端：2 个真实系统工作伴侣、6 个皮肤和 20 个动画状态。
- 移动端：详情抽屉、28 个动作资源、关闭交互和底部导航避让通过。
- 桌面与移动端均无横向溢出。
- 付费模型调用：`0`。
