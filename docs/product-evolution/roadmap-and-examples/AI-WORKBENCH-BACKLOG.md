# AI 编程工作台后续 Backlog

本文记录暂不在当前轮实现、但后续值得继续推进的方向。

## P0：真实开发 E2E 验收

准备 2-3 个真实小项目，从群聊一句自然语言需求开始，完整验证：

- 主 Agent 是否正确判断是否需要协调。
- 是否派发到正确项目 Agent。
- 子 Agent 是否真实修改代码。
- 是否展示代码 diff。
- 是否运行验证。
- 失败后是否返工。
- 最终报告用户是否看得懂。

## P1：记忆召回质量评估

项目记忆、群聊记忆、全局记忆已可用，下一步应评估召回质量：

- 是否召回了有用历史。
- 是否漏掉关键结论。
- 是否召回过期信息。
- 简报是否过长。
- 记忆和当前代码冲突时是否以当前代码为准。

## P1：权限和安全边界增强

继续明确哪些操作必须确认：

- 删除、清空、重置、迁移、部署、数据库、支付、密钥、生产环境操作。
- 子 Agent 不得越权改其他项目。
- Cursor / Codex / Claude 权限模式需要统一展示。

## P2：产品化设置和调试面板

后续可增加：

- Agent 在线状态。
- 每个项目当前执行器。
- Claude / Codex / Cursor 可用性。
- 最近失败原因。
- 当前任务队列。
- 当前项目锁。
- 当前记忆状态。
- 一键清理卡住任务。
