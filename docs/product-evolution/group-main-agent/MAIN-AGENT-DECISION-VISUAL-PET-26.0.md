# 主 Agent 决策可视化 + 宠物状态联动 2.0

本轮目标是让群聊主 Agent 更像一个可理解的 AI 编程协调者：用户能在群聊顶部看到主 Agent 最近一次判断，也能通过桌面宠物/工作台宠物看到它当前是在思考、计划、派发、等待确认还是普通回复。

## 已完成

- 群聊顶部新增“最近决策”状态。
  - 展示决策模式：普通回复、项目分析、创建任务、派发子 Agent、追加要求、任务治理。
  - 展示动作链摘要：读群聊、读代码、查知识库、建任务、派发、读回执、重规划、回复等。
  - 展示下一步原因，并支持一键定位到对应消息里的完整决策卡。

- 主 Agent 决策卡补充用户可读说明。
  - 普通对话会说明“没有派发：这轮是普通对话”。
  - 项目分析会说明“没有派发：只读项目分析，不修改项目”。
  - 明确任务会说明已创建任务/已派发。
  - 高风险治理动作会提示需要用户确认。

- 技术详情保留但默认折叠。
  - 用户默认只看到简单状态。
  - 展开后可按 Action 理解本轮读取群聊、读取代码快照、查询知识库、创建任务、派发子 Agent、读取回执、验收和最终回复。

- 宠物状态接入真实后端状态流。
  - `conversation` → thinking：正在组织回复。
  - `project_analysis` → thinking：只读查看项目上下文。
  - `project_task` → planning：已创建任务，准备安排执行。
  - `delegation` → building：正在派发子 Agent 协作。
  - `followup` → working：追加要求并入原任务。
  - `governance` 或需要确认 → waiting：等待明确确认。
  - 决策状态会同步给协调者宠物，也会显式同步给 `global-agent`，避免只在内部 Agent 上更新而用户看不到。

- 清理了前端无监听的 `ccm-pet-speech` 本地事件。
  - 宠物联动统一走 `/api/status/stream`，桌面宠物和工作台宠物使用同一套状态。

## 自测覆盖

脚本：`scripts/main-agent-decision-ui-selftest.mjs`

检查内容：

- 决策卡存在并显示读取内容、本轮动作、权限判断、下一步。
- 决策卡包含“没有派发/需要确认/已创建/已派发”等解释。
- 技术详情包含 Action trace。
- 群聊能接收 `main_agent_decision` SSE 并写入消息。
- 群聊顶部能展示最近一次决策，并可定位到消息。
- 任务体验卡仍能展示主 Agent 决策。
- 后端会把 `main_agent_decision` 随任务创建返回。
- 后端会把主 Agent 决策同步到宠物状态流和 `global-agent`。
