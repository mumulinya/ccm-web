# 对话搜索来源筛选栏优化

Date: 2026-07-20

## 问题

原来的“全部 / 全局助手 / 群聊 / 项目 / 飞书”被铺成一条贯穿页面的导航栏，但这些选项本质上只是搜索来源筛选。它占用了过强的视觉层级，“收藏”又被推到页面最右侧，来源与视图切换关系不清晰。

## 调整

- 将来源标签收进搜索标题区域，改为紧凑分段控件。
- 每个来源使用一致的 Lucide 图标，选中态改为轻量表面高亮，不再使用整条导航下划线。
- 收藏作为相邻但独立的视图按钮，不再漂在页面最右侧。
- 搜索后缓存并显示各来源结果数量；来源重新搜索时更新对应数量。
- 从收藏点击任一来源会自动返回搜索结果视图，修复此前只改变来源却仍停留在收藏页的问题。
- 移动端隐藏来源图标和数量，保留完整文字，避免 390px 宽度下裁切或横向页面溢出。

## 验证

- `npm run test:conversation-search`
- `npm run build:frontend`
- Playwright 桌面端 `1536 x 682`：运行时错误为 `0`。
- Playwright 移动端 `390 x 844`：来源与收藏完整显示，无页面横向溢出。
