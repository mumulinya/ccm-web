# 全局 Agent 与音乐 Agent 语义播放升级

Date: 2026-07-20

## 目标

让用户不必每次提供精确歌名。全局 Agent 和音乐 Agent 都能理解“心情不好”“今天很开心”“适合写代码”“播放某位歌手的歌”一类请求，并通过同一播放器从本地曲库或外部音乐 API 选择歌曲。

## 完成内容

- 新增 `ccm-music-playback-plan-v1`，区分精确歌曲、歌手随机、情绪推荐、曲风推荐和完全随机。
- 播放规划优先调用统一大模型；模型不可用时使用确定性语义兜底。
- 全局 Agent、client effect、远程队列和后台轮询全程保留完整 `request_text`。
- 音乐 Agent 自动播放时传递当前用户完整消息，不再只传清洗后的关键词。
- 精确歌曲保留原有严格候选分数；推荐请求使用模型语义选择，不会被精确歌名门禁误拒绝。
- 歌手请求会先核验候选歌手字段，再由大模型结合用户原话从匹配作品中选择；只有模型不可用时才随机兜底。
- 通用随机仍保持本地优先、外部 API 兜底，并继续遵守最后一次播放请求生效的规则。

## 失败策略

- 模型意图规划失败：使用后端兜底计划。
- 本地没有相关歌曲：按用户当前音乐模式查询外部 API。
- 歌手请求没有歌手匹配候选：拒绝错误播放，不从无关候选随机。
- 精确歌名没有足够匹配：保持拒绝，不以推荐模式绕过。
- 新播放请求到达：旧的未完成播放被 supersede，页面不会同时播放两首歌。
