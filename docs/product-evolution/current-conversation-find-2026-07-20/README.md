# 当前会话内搜索

Date: 2026-07-20

## 目标

在用户已经进入一个具体会话后，允许直接搜索该会话中的用户消息、Agent 回复和结构化消息内容，不必跳转到跨会话“对话搜索”页面。

## 统一交互

- 标题栏放大镜按钮打开当前会话搜索。
- Windows/Linux 使用 `Ctrl+F`，macOS 使用 `Command+F`。
- 输入时实时显示“当前匹配 / 总匹配”数量。
- `Enter` 跳到下一条，`Shift+Enter` 跳到上一条。
- 上下箭头按钮可以循环浏览所有匹配。
- `Esc` 或关闭按钮退出搜索并移除匹配高亮。
- 匹配消息使用黄色边框，当前结果使用更明显的背景和边框。

## 作用范围

| 页面 | 搜索边界 |
| --- | --- |
| 项目 Agent | 当前项目下当前精确项目会话 |
| 群聊主 Agent | 当前群聊下当前 `gcs_*` 会话，只搜索用户可见消息 |
| 全局 Agent | 当前精确全局会话 |
| 音乐助手 | 当前单例音乐助手 transcript，不新增会话 |

切换项目、群聊或全局会话会关闭搜索并清空关键词。隐藏页面不会响应 `Ctrl+F`，音乐助手抽屉关闭时也不会抢占浏览器快捷键。

## 实现边界

- 搜索直接读取当前页面已经加载的权威消息数组，不创建搜索副本或新数据库。
- 搜索不会修改 transcript、摘要、长期记忆或压缩边界。
- 结构化对象采用文本字段递归提取，不使用 `v-html`，避免搜索高亮引入 HTML 执行风险。
- 会话内搜索与跨会话“对话搜索”并存：前者用于当前位置快速定位，后者用于跨来源检索。

## 验证

- `npm run test:conversation-find`
- `npm run test:message-navigator`
- `npm run build:frontend`
- Playwright 全局会话：`Ctrl+F`、匹配计数和 `msg-0` 高亮通过。
- Playwright 群聊和音乐助手：标题栏入口可见，当前页仅有一个有效搜索入口。
- 测试付费 Provider 调用为 `0`。
