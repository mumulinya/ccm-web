# 音乐助手侧边抽屉

## 目标

音乐助手不再常驻占据播放器右栏。页面默认显示完整的封面、歌词和曲库，只在用户点击右侧音乐助手按钮时展开助手，再次点击同一按钮或抽屉内收起按钮即可关闭。

## 行为

- 音乐助手默认关闭，不创建可交互的隐藏聊天面板。
- 页面右侧保留竖向“音乐助手”按钮，并展示展开状态。
- 点击侧边按钮打开或关闭抽屉。
- 顶部“音乐助手”入口控制同一个抽屉，不维护第二套状态。
- 抽屉打开后自动聚焦助手输入框。
- 抽屉右上角提供明确的收起按钮。
- 切换到正在播放、曲库、歌单或队列时自动收起助手。
- 对话历史、搜索结果和 Agent 状态继续由原有 composable 保存，收起不会清空内容。

## 布局

- 关闭时播放器使用两栏播放舞台，曲库横跨两栏。
- 打开时抽屉固定在视口右侧，不参与网格宽度计算，不压缩封面和曲库。
- 桌面抽屉宽度不超过 420px；手机抽屉保留侧边开关并占据可用宽度。
- 抽屉打开后页面 `scrollX` 必须保持为 0，禁止产生横向空白区域。

## 验证

- `npm run build:frontend`
- `CCM_MUSIC_URL=http://127.0.0.1:3080/ npm run test:music-render`
- Playwright 验证默认关闭、顶部入口开关、侧边按钮重复开关、桌面无横向位移、手机展开/收起以及原有歌单、下载和助手业务流程。
