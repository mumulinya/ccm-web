# 内置宠物动作预览修复

## 问题

- 宠物空间的动作按钮会更新文字状态，但内置宠物仍使用静态图片，画面不会变化。
- 内置宠物进入状态渲染器后，Clawd 被按通用文件名读取，实际资源却位于 `pets/clawd/` 专属目录，导致动作图片加载失败。
- 旧错误回退会把任何加载失败的宠物替换成月薪喵，造成宠物名称与画面不一致。

## 修复

- 六种内置宠物统一进入状态感知渲染器。
- Clawd 的 20 个状态使用现有专属资源映射；例如待机读取 `clawd-idle-follow.svg`，规划读取 `clawd-working-ultrathink.svg`。
- 图片加载失败时只回退到当前宠物的待机或基础形象，不再替换成其他宠物。
- 状态图片使用资源 URL 作为渲染键，切换动作时创建干净的图片节点，避免旧错误状态污染下一次预览。

## 验证

- `node scripts/pet-built-in-action-preview-selftest.mjs`
- `npm run build:frontend`
- 浏览器点击 Clawd 的待机、规划和工作状态，确认资源 URL 与画面随状态变化。
- 未修改宠物配置、应用关系或用户数据。
- 付费模型调用：`0`。
