## 使用规则

1. 判断当前系统行为时，先看 [当前状态](../CURRENT.md) 和 [确认项目结构](../confirmed-project-architecture/README.md)。
2. 排查某次升级的实现理由、测试截图或兼容决策时，再进入历史目录及其 `CATALOG.md`。
3. 历史文档中的待办和结论不会自动覆盖当前确认文档。
4. 不批量删除或移动历史记录；确需废弃时，在当前文档中标注替代关系。
