# Marketplace Custom Source Autosave Install

## Why

Marketplace install/update is now source-bound: the backend only installs canonical items that it can reload from a trusted source. That closes the client-payload tampering gap, but it also means custom URL items cannot be installed directly from an unsaved source.

This upgrade keeps the marketplace workflow ergonomic. When a user loads an external HTTPS catalog, GitHub Skill directory, or `SKILL.md` URL and clicks install/update, the frontend first saves that URL as an auditable marketplace source, reloads the canonical item from the saved source, then submits a minimal source-bound install request.

## Changes

- `frontend/src/api/index.js`
  - Encodes the marketplace `source` query parameter.

- `frontend/src/components/tools/ToolsConfig.vue`
  - Tracks the URL and label used for the currently loaded custom source.
  - Marks items loaded from unsaved custom URLs with `sourceNeedsSave`.
  - Adds a source gate note explaining that install will first save the source and then let the backend re-read/verify the item.
  - Adds `marketplaceInstallPayload`, so install/update submits only `id/type/name/source`.
  - Adds `saveCustomSourceAndInstall`, which:
    - saves the custom URL as a marketplace source;
    - reloads marketplace items from that saved source;
    - matches the canonical item by `id` or `type:name`;
    - calls install/update using the source-bound payload.
  - Avoids noisy backend errors when the custom source selector is active but no URL has been entered.

## Risks

- Installing from a custom URL now creates a saved source record before installation. This is intentional so future updates/uninstalls have an auditable source trail.
- If a source changes between the initial custom URL load and the autosave install, CCM installs the canonical item fetched after saving, not stale client-side material.
