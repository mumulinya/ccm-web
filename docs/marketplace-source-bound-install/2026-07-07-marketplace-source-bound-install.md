# Marketplace Source-Bound Install

## Why

CCM 工具商城已经支持外部 catalog、GitHub Skill、Claude plugin marketplace 转换、预览、安装、更新和卸载。但公开安装接口此前直接信任前端提交的完整 marketplace item。即使列表和保存来源阶段做了 HTTPS、内网地址和来源 trust 约束，安装阶段仍可能被篡改 payload，导致后端安装未经 catalog 证明的 command、env、url 或 Skill 下载地址。

这次升级把 marketplace 安装/更新改为 source-bound：公开 API 只把客户端 payload 当作 item 标识和来源线索，真正安装前由后端根据 source 重新加载内置来源、Smithery、CCM community 或已保存外部来源，并只安装后端解析到的 canonical item。

## Changes

- `backend/modules/tools/marketplace.ts`
  - 新增 `CCM_COMMUNITY_CATALOG_URL` 常量，避免 community catalog URL 在列表和安装复验中分叉。
  - 新增 `normalizeMarketplaceInstallRequest`，安装/更新请求只需要 `id/type/name/source`，不把客户端提交的 command/url/prompt 当作可信材料。
  - 新增 `loadMarketplaceItemsForSource`，支持从以下可信来源复验 item：
    - `ccm-official` 内置商城；
    - `smithery`；
    - `ccm-community`；
    - 用户已保存并启用的外部 HTTPS catalog/GitHub/Direct Skill 来源。
  - 新增 `resolveMarketplaceItemForInstall`，安装/更新前按 source 重新解析 item，按 `id` 或 `type:name` 匹配 canonical item。
  - `/api/marketplace/install` 与 `/api/marketplace/update` 现在通过 source-bound resolver 后再写入 MCP/Skill catalog。
  - 未保存的 custom source 不能直接安装，必须先保存为 CCM marketplace source。
  - 自测新增：
    - 篡改后的 install payload 不会覆盖 catalog 中的 canonical command/args；
    - 未保存外部来源会被拒绝；
    - update 同样使用 catalog 重新解析出的版本和材料。

## Risks

- Direct install from an unsaved custom URL is now rejected. Users should save the external source first, then install from that source. This is intentional: marketplace install should be bound to an auditable source record.
- If a source catalog removes or renames an item, update will fail instead of applying stale client-submitted material.
