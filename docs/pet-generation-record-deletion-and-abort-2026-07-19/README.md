# 宠物生成中断说明与记录删除

## 原因

页面中的两条同名“虎子”记录是两个独立生成任务。第一条来自原任务重做，第二条来自重新提交；服务不会按名称覆盖任务，因此都会保留。

两条记录的 `This operation was aborted` 都表示模型请求执行期间被中断。本次发生在服务重新构建和重启期间，不是参考图读取失败，也不是相似度门禁给出的结论。

## 修改

- 将 AbortError、`This operation was aborted` 和同类请求中断错误统一显示为中文：生成请求可能因服务重启、网络中断或模型超时而中断，可以重试。
- 旧记录中的英文错误在 API 读取时也会惰性转换，不需要批量改写历史 JSON。
- 失败、已取消和已完成的生成记录增加“删除记录”按钮。
- 旧版 Codex 图集历史也支持逐条删除。
- 正在排队、生成、确认或安装的任务必须先取消，之后才能删除。
- 删除会移除任务记录，并且仅在任务目录经过 `pet-generation` 根目录边界校验后清理临时参考图、候选 SVG、渲染图和 QA 文件。
- 删除已完成记录不会删除已经复制到正式宠物资源目录中的已安装宠物。

## 验证

- 后端与前端 production build：通过。
- 全仓 TypeScript check：通过。
- SVG 宠物 mock self-test：通过，验证删除 API 和中断错误本地化已接入。
- 宠物空间桌面/移动端渲染回归：通过，验证终态记录展示删除入口。
- 自动测试付费 Provider 调用为 `0`。

现有两条失败记录不会被自动删除。用户可在刷新后的生成记录中逐条删除，避免系统替用户做不可逆的数据清理决定。
