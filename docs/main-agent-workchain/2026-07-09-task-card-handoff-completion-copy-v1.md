# 任务卡完成态 handoff 文案收紧 v1

## 背景

统一任务卡同时服务全局主 agent、群聊主 agent 和项目执行结果。旧文案里：

- 全局运行完成态的下一步兜底是“任务已处理完成”。
- 项目执行成员通过验收后的交付标题是“项目执行成员已完成本轮处理”。

这些文案不够具体，容易把“执行成员回传/任务卡处理完”误读成最终交付已经无条件完成。

## 本次调整

- 完成态下一步改为“可以查看交付总结、验证结果和风险提示”。
- 项目执行成员强验收通过时，标题改为“项目执行成员已提交可验收结果”。
- 弱验收项目仍显示“仍需补齐验证或验收”，不允许只靠 `done` 状态通过。
- `test:chat-experience` 增加行为断言，静态自检禁止旧文案回归。

## 验收口径

任务卡完成态必须把用户引向交付总结、验证结果和风险提示；执行成员回传结果只是进入主 agent 验收链路，不能替代最终总结。
