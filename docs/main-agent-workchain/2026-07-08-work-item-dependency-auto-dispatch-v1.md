# 工作项依赖解锁自动接续 v1

## 目标

上一版已经能告诉用户工作项为什么暂未派发。本版补齐下一段链路：当前置工作完成后，主 Agent 不只是显示“可派发”，还会记录哪些后续工作项被解锁，并在安全条件满足时自动接上下一步派发。

## 本次实现

- 新增 `ccm-main-agent-work-item-unlock-summary-v1` 用户可见摘要。
- 后端在子 Agent 结果说明写回工作项时比较前后依赖状态，识别刚刚解锁的工作项。
- 后端 `next_claimable` 现在纳入“依赖已完成的 blocked 工作项”，避免前端显示可派发、后端不认的分裂。
- 如果当前任务允许自动执行且调用现场有 `ctx`，主 Agent 会用现有 `continueTaskWithMessage` 接上 `next_claimable_work_item`。
- 如果当前执行轮还在运行，自动接续会进入内部 pending followup；本轮结束后由原有队列机制继续派发。
- 自动接续不会向群聊追加一条伪装成用户的“任务补充说明”，只更新任务状态和任务卡。
- 解锁的工作项 ID、内部 auto dispatch 结果保留在技术详情；用户区只显示“前置完成、已自动接上、下一步”。

## 用户可见行为

- 群聊任务卡和全局任务卡会显示“前置完成，下一步已解锁”。
- 如果已自动接上，会显示“已自动接上”以及“当前执行轮结束后继续派发”。
- 如果执行通道暂时不可用，会显示等待接续的友好说明。
- 普通问话仍不会展示工作项、Todo 或技术字段。

## 回归覆盖

- 后端工作项自测覆盖 `buildMainAgentWorkItemUnlockSummary` 和新解锁项。
- 静态 UI 自测覆盖后端自动接续、前端解锁卡片、群聊/全局数据透传。
- Playwright 渲染回归覆盖群聊与全局任务卡：
  - 解锁摘要可见。
  - “已自动接上”可见。
  - 内部字段和工作项 ID 不出现在用户区。
