# CCM 本地登录与注册

Date: 2026-07-22

## 目的

CCM 现在默认要求浏览器先登录，再访问项目、群聊、全局 Agent、音乐和设置等业务 API。登录页是应用的入口状态，不改变后端 Agent、会话记忆和项目记忆的业务边界。

## 初始账户

首次启动且尚未存在本地账户时，系统自动创建一个管理员账户：

- 用户名：`mumulin`
- 初始密码：由本次部署要求设置，首次登录后建议立即在“系统设置 -> 账户与安全”修改。

密码不会以明文写入文件，也不会通过 API 返回。

## 注册策略

- 注册默认关闭。
- 首次安装是唯一例外：登录页会显示注册入口，并提示用户可以使用初始管理员登录或创建普通账户；首次成功登录/注册后即结束引导，注册恢复为关闭。
- 管理员登录后，在“系统设置 -> 账户与安全”打开“注册入口”，未登录用户才会看到注册选项。
- 注册只创建普通账户，不授予管理员设置、Agent 凭据或其他账户管理权限。
- 管理员可以随时关闭注册；关闭后已有账户仍可登录。

## 请求流程

```text
浏览器打开 CCM
  -> GET /api/auth/session
  -> 未登录：展示登录页
  -> 登录成功：签发 ccm_session HttpOnly Cookie
  -> Root 挂载工作台 App
  -> 浏览器业务 API 需要同源且具有有效会话
```

公开接口只有会话状态、登录和注册入口。登录后的账户设置包括：

- `GET /api/auth/settings`：管理员读取注册开关和账户数量；
- `PUT /api/auth/settings`：管理员开启或关闭注册；
- `POST /api/auth/password`：当前用户校验旧密码并修改新密码；
- `POST /api/auth/logout`：删除当前会话。

## 存储与安全

数据位于当前用户目录下的 `.cc-connect/auth/`：

- `users.json`：账户、角色、注册策略和 scrypt 密码哈希；
- `sessions.json`：只保存 SHA-256 会话令牌哈希，不保存浏览器 Cookie 原文。

账户文件使用原子写入和文件锁，目录权限为 `0700`，文件权限为 `0600`。Cookie 使用 `HttpOnly`、`SameSite=Strict`、`Path=/` 和七天过期时间；密码修改会使该用户所有旧会话失效。登录失败按来源和用户名限流。

浏览器业务 API 同时检查同源请求和有效会话。无浏览器来源标记的本地 Node/Provider/回归脚本请求保持兼容，用于 CCM 内部服务调用；这不是把认证绕过暴露给浏览器。

