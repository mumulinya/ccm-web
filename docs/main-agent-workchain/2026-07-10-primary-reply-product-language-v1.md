# 主 Agent 主文本产品化表达 v1

## 背景

全局主 Agent 在规则兜底派发、派发进度和群聊流式失败时，仍有少量主文本使用“群聊主 Agent”“项目 Agent”“下游 Agent”“错误：”等实现术语。即使前端能够清洗部分旧数据，新生成的数据也应该从源头就是用户能理解的表达。

## 本次升级

- 全局直派群聊改为“交给协作群继续拆分和跟进”。
- 全局直派项目改为“交给项目的执行成员”。
- 跨项目需求改为“建立跨项目执行计划，并交给执行目标持续跟进”。
- 派发进度中的角色统一为“协作群 / 项目执行成员”，失败状态统一为“待排查”。
- 全局 Todo 兜底不再显示“下游 Agent”，改为“执行目标”。
- 群聊流式失败改为“这次没有完成：……”，并说明会保留当前进度；技术排障信息继续放入技术详情。
- 协作问答和缺失验证来源的默认成员名统一为“执行成员”。
- 展示层会精确移除旧角色名替换后产生的残留空格，避免出现“我 判断”“协作群 接管”，同时保留“必检项 命令验证”一类有意义的业务分隔。

## 兼容策略

历史任务中已有的“全局主 Agent / 群聊主 Agent / 下游 Agent”仍由展示层清洗，因此旧任务和新任务会得到一致的用户可见结果。

## 验证范围

- 全局意图与直派自测验证群聊、项目和跨项目回复不再暴露内部角色名。
- 源码守卫验证群聊错误消息、全局 Todo 和默认成员名。
- Playwright 使用包含旧术语的历史 fixture，验证真实渲染只显示“协作群 / 执行目标”。
- 类型检查、前后端构建和完整渲染回归。

## 验证结果

- 全局意图与直派自测：通过，群聊、项目和跨项目三类回复均使用产品语言。
- 主 Agent 决策 UI 源码自测：通过。
- `npm run check`、`npm run build:backend`、`npm run build:frontend`：通过。
- `npm run test:render-regression`：通过，共 23 张真实截图。
- 渲染回归保留旧历史术语作为输入，并确认用户界面不显示“全局主 Agent / 群聊主 Agent / 下游 Agent”。
