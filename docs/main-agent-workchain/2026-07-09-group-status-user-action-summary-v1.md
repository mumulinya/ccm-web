# Group Status User Action Summary v1

## 背景

用户在群聊里问“现在进展怎么样”时，主 Agent 不应该只说任务正在处理。如果当前卡在计划确认、补充信息、待处理问答或阻塞项上，用户需要在普通文本回复里直接看到自己下一步要做什么。

## 本次修改

- 群聊状态追问新增“需要你处理”摘要。
- 从当前 Todo、等待用户阶段、开放问答、阻塞项和剩余需求中提取用户可执行的下一步。
- 状态追问仍然不创建新的 Todo，也不展示任务卡，只读取已有状态并回答。
- `CCM_AGENT_RECEIPT`、`trace_id` 等内部协议仍然不会出现在用户可见文本里。

## 验证

- 运行时自测覆盖计划待确认场景。
- 静态自测覆盖 `buildGroupStatusUserActionSummary` 接线。
- 既有群聊状态追问、进度刷新、执行成员等待情况和协议隐藏断言继续保留。
