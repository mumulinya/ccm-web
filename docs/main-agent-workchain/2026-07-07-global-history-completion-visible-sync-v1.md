# 全局 Agent 最终总结可见同步 V1

## 背景

全局 Agent 直接把任务派发给群聊主 Agent 后，后端已经可以在群聊任务通过验收时，把最终总结写回 `global-agent-history.json`。

但前端全局会话主要读取浏览器 localStorage。如果用户页面没有刷新，或者 localStorage 比后端旧，最终总结可能只存在后端文件里，用户在全局 Agent 对话里看不到，甚至可能被旧前端历史覆盖。

## 本次升级

- 后端 `/api/global-agent/history` 保存逻辑从“前端覆盖后端”改为“按会话合并消息”。
- 后端已有的“群聊任务通过验收最终总结”会被保留，不会被旧 localStorage 冲掉。
- 合并时会先去重和按时间排序，再截断历史长度；即使前端一次性回传旧的 80 条消息，也不会挤掉后端刚写入的完成总结。
- 全局 Agent 前端启动时会从后端拉取历史并合并新增消息。
- 全局 Agent 页面保持轻量轮询，每 5 秒检查一次后端是否有新总结。
- 同步只更新用户可见的友好总结；Trace、运行 ID、内部回执和协议细节仍放在任务卡技术详情里。

## 对用户体验的影响

- 用户从全局 Agent 派发群聊任务后，不会只看到“已派发”。
- 群聊主 Agent 真正完成并通过验收后，全局 Agent 会话会自动出现最终总结。
- 普通问话不受影响，不会因为这次同步机制展示 Todo 或任务卡。
