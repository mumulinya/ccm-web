# 主 Agent 接续状态闭环 v1

## 背景

参考 Claude Code 的任务链路后，本项目已经具备计划、执行队列、回执质量和精准返工能力，但用户在点击“继续派发”或“精准返工”后，任务卡以前主要只显示 toast，缺少可持续感知的“主 Agent 已接上这一轮”的状态。

## 本次升级

- 群聊任务续跑会写入 `collaboration_state.last_continuation` 和 `continuation_events`，记录来源、目标、原因、返工类型和工作项 ID。
- 任务卡新增 `continuation_status`，用用户可读语言展示“下一步派发已接上 / 精准返工已接上 / 补充要求已接收”。
- 同一条接续会写入用户可见时间线，普通用户看到的是进展状态，完整返工指令继续留在技术详情或接口返回中。
- 全局任务卡兼容同一结构；从全局任务卡点击“继续派发”时，会把工作项元数据传给全局监工并保存。

## 用户可见效果

- 用户能看到主 Agent 已经接收了哪个目标的下一步工作项。
- 用户能看到当前是已入队、处理中，还是本轮结束后继续。
- 技术字段、原始返工指令、trace/session 信息不会直接出现在普通文本框里。

## 验证

- 增加静态自测覆盖：后端输出、前端统一任务卡、全局任务卡动作传参。
- 增加统一任务体验自测覆盖：全局任务可显示接续状态。
- 增加 Playwright 渲染断言：真实页面截图回归中校验“下一步派发已接上”可见。
