# 群聊普通回复简洁展示 V1

## 问题

群聊主 Agent 已将问候等内容判定为普通对话，但共享决策卡仍显示“处理方式、下一步、工具摘要、处理链路和技术详情”。这与全局 Agent 的简洁问答体验不一致。

## 修复

- 在共享的 `MainAgentDecisionCard` 组件层识别纯普通对话。
- 普通问候、问答且无任务动作时，只展示 Agent 的自然语言回复。
- 不再展示 Todo、决策摘要、工具摘要、处理链路和技术详情。
- 待用户确认、权限阻塞、活跃步骤、创建任务和派发执行等非普通对话仍显示决策卡。

## 验收

- 群聊输入“你好”时，只显示协调者的回复正文。
- 全局 Agent 和群聊主 Agent 共享同一套普通对话决策卡隐藏规则。
- 需要用户操作或真实任务执行时，可见状态与任务卡不受影响。

## 验证结果

- `node scripts/ordinary-reply-visibility-selftest.mjs` 通过，7 个普通回复、澄清、阻塞和真实任务场景均符合预期。
- `npm run build:frontend` 通过，Vite 成功构建 188 个模块。
- 运行服务继续由 `http://127.0.0.1:3080/` 提供，并已加载本次新生成的前端资源。
