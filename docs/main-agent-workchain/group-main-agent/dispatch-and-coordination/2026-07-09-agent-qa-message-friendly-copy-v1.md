# 协作问答消息用户友好文案

日期：2026-07-09

## 背景

群聊里的协作问答消息直接展示在聊天流中。原先组件会显示“目标 Agent”“原 Agent”“Agent 问答进展”等内部术语，用户需要额外理解系统实现角色。

## 本次调整

- 将问答状态改为用户可懂的成员语义：
  - “等待目标 Agent” -> “等待目标成员”
  - “目标 Agent 工作中” -> “目标成员处理中”
  - “已注入原 Agent” -> “已交回提问方”
  - “原 Agent 已续跑” -> “提问方已继续处理”
- 将 fallback 摘要改为“协作问答进展已更新”。
- 将人工操作文案从“人工接管”改为“人工接手”。
- 群聊操作反馈从“Agent 问答”改为“协作问答”。
- 新增真实渲染 fixture，验证协作问答消息默认不展示执行 ID、路由策略等技术详情。

## 验收点

- 用户默认看到的是提问方、目标成员、问题和下一步。
- 执行 ID、路由、权限模式、证据等仍默认折叠在技术详情里。
- 群聊任务卡和全局/群聊主 Agent 的协作链路展示继续保持用户可读。
