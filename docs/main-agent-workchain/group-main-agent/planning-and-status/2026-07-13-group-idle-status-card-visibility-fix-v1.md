# 群聊空闲协作状态框隐藏修复 V1

## 问题

群聊顶部协作状态框只要能读取到“最近决策”就会渲染。普通问答也会生成一条 `conversation` 决策，因此任务已为空闲时仍显示“协作状态 / 执行中无 / 待确认 0”的空面板。

## 修复

- 新增群聊协作状态可见性判定器。
- 普通回复、空闲状态和已完成且无待处理事项的状态不再显示顶部协作框。
- 仅在存在活跃任务、执行成员、待确认问答、阻塞项、活跃 Todo，或建任务/派发/治理等工作流决策时显示。
- 保留任务执行期间的实时状态、待确认和阻塞提示。

## 验收

- 普通问话后不显示协作状态框。
- 无任务的空闲群聊不显示空面板。
- 任务执行、待确认、阻塞和派发决策仍正常显示。

## 验证结果

- `node scripts/group-status-visibility-selftest.mjs` 通过，10 个空闲、任务、问答和阻塞场景全部符合预期。
- `npm run build:frontend` 通过，Vite 成功构建 188 个模块。
- 运行服务 `http://127.0.0.1:3080/` 返回 HTTP 200，已加载新的主资源和群聊分包。
