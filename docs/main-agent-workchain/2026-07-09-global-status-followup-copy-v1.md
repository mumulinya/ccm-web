# 全局状态跟进文案 v1

日期：2026-07-09

## 背景

用户问“现在进展怎么样”“任务做到哪了”时，看到的应该是可理解的任务状态、阻塞、下一步和验收口径。旧文案里部分全局状态摘要使用“全局监工/监工中/监工状态”等内部运维词，容易让用户感到是在看系统实现，而不是任务进展。

## 改动

- 全局状态摘要中的“全局监工”改为“持续跟进”。
- 全局流式事件的“监工中”改为“持续跟进中”。
- 工具显示名从“查询监工状态/管理异步监工”改为“查询持续跟进状态/管理持续跟进”。
- 历史进展文本通过展示层 sanitizer 兼容清洗：旧的“全局监工已安排返工”会显示为“全局任务跟进已安排返工”。

## 回归保护

- 静态守卫要求全局状态摘要包含“持续跟进”，并禁止用户可见状态行恢复为“全局监工：”。
- 渲染回归检查任务进展里显示“全局任务跟进已安排返工”，并确认默认可见区域不显示“全局监工”。

## 预期体验

用户看到的是“我正在持续跟进执行、验收和最终总结”“全局任务跟进已安排返工”等任务语言；内部 supervision 机制仍保留在技术结构和 API 名称里，便于系统管理与排障。
