# 全局 Agent 动作结果展示文案 v1

本轮目标：继续对齐 `D:\claude-code` 的用户可见输出原则。全局主 Agent 执行动作后，用户应该看到“处理结果”和下一步状态，而不是“系统回执”这种偏内部协议的说法。

## 改动

- 全局主 Agent 新生成的动作结果统一使用 `[处理结果]` 标记。
- 音乐、桌宠、页面跳转、协作任务、群聊派发、定时任务、项目控制、项目创建和模板创建等动作都通过统一的 `systemResultMessage()` 输出。
- 旧历史消息里的 `[系统回执]` 仍保留兼容解析，避免用户以前的会话卡片失效。
- 源码注释和卡片说明改成“处理结果卡片”，降低后续维护时继续误用“系统回执”的概率。
- 静态自测新增护栏：除历史兼容常量外，`GlobalAgent.vue` 里不能再残留新展示用的 `[系统回执]` 或“主 Agent 回执”文案。

## 用户感知

- 用户看到的是“协作开发任务已派发”“定时开发任务已配置”“项目已启动”等可理解结果。
- 旧会话不会坏；旧 `[系统回执]` 仍能渲染成同一类高阶卡片。
- 技术协议、回执结构和 trace 信息继续留在技术详情或后端内部链路，不进入普通正文。

## 验证

- `scripts/main-agent-decision-ui-selftest.mjs` 覆盖全局动作结果文案护栏。
- 后续运行前端类型检查、构建和渲染/回放回归，确认该文案调整不影响任务卡、技术详情和普通问话展示。
