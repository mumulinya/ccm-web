# 全局工具观察返回文案 v1

日期：2026-07-09

## 背景

全局主流程里，工具调用、飞书管理动作、项目直派和群聊直派会先返回一段观察结果。这个“返回”只说明底层动作已经有结果，不代表用户需求已经完成。旧文案里部分 fallback 会写成“操作已完成”或“Agent 已返回执行结果”，容易和最终交付完成混淆。

## 改动

- 全局直派执行结果 fallback 改为“执行目标已返回结果”，避免把执行器回传误读成最终交付。
- 工具观察摘要从“操作已完成”改为“操作已返回结果 / 查询已返回”，并继续提示详细记录在技术详情。
- 群聊直派接管摘要中的“主 Agent 说明”改为“协作说明”，默认用户可见总结不再露内部角色标签。
- 项目直派摘要中的“项目 Agent”改为“项目执行成员”。

## 回归保护

- 后端 intent/selftest 现在要求工具观察 fallback 包含“已返回结果/查询已返回”，并禁止旧的“操作已完成；详细记录已放入技术详情。”。
- 静态守卫要求直派摘要使用“协作说明”，并禁止“主 Agent 说明 / 主 Agent 回执”出现在用户可见摘要路径。

## 预期体验

用户会看到“结果已经返回，我正在整理/检查”的状态；只有通过任务卡验收、验证和最终总结后，才会看到真正的完成口径。
