# Global Status Supervision Summary v1

## 背景

用户问“现在进展怎么样/还差什么”时，应该看到已有任务的真实状态，但不应该创建新的 Todo，也不应该看到 `trace_id`、`global_run_id`、`supervisor_id` 等内部协议字段。

## 本次修改

- 全局状态追问会读取关联的全局监工运行记录。
- 当监工处于等待用户处理、阻塞、暂停状态时，用户可见摘要显示“等你处理阻塞点，这还不是完成结果”一类明确提示。
- 当监工处于返工状态时，用户可见摘要显示“正在返工，修复后重新运行 TestAgent/独立复核”。
- `waiting_user`/`paused` 状态在用户摘要里归类为“需要处理”。
- 状态摘要继续保持文本型回复，不为普通进展追问创建新的任务 Todo。
- 全局直派接管说明统一使用“主 Agent 说明”，避免继续出现旧式回执标签。

## 验证

- 自测覆盖全局状态追问识别。
- 自测覆盖执行成员等待状态展示。
- 自测覆盖全局监工等待用户处理与返工状态展示。
- 自测覆盖内部协议和运行标识不出现在用户可见摘要中。
