# 全局直派任务安全撤销回写 V1

## 背景

对照 `D:\claude-code` 的 checkpoint / rewind / resume 思路后，发现本项目已经支持群聊任务检查点和安全撤销，但全局主 Agent 直派到群聊主 Agent 的任务在“完成后又撤销”时，缺少一条完整回写链路：全局会话和长期记忆可能仍保留“已通过验收”的历史结论。

## 本轮升级

- 群聊任务执行安全撤销后，会自动向对应全局 Agent 会话追加一条用户可读的撤销总结。
- 撤销总结只展示结论、影响和下一步，不把 `trace_id`、`global_run_id`、回执协议等内部内容放到用户可见文本。
- 全局长期记忆会移除同一个直派任务原先的“已通过验收”结论，并写入“已安全撤销，不再视为完成或已交付”的新结论。
- 任务 timeline 增加 `global_direct_dispatch_rollback_synced` 事件，技术详情仍可追踪同步证据。

## 验收点

- 已补 `runCollaborationUxSelfTest()`：检查撤销同步条件、重复通知保护、用户可见撤销文案不泄露内部字段。
- 已补 `runGlobalAgentMemorySelfTest()`：检查撤销记忆会覆盖完成记忆，后续追问时返回“已安全撤销”而不是继续当成完成。
- 已补 `scripts/main-agent-decision-ui-selftest.mjs` 静态守护，防止后续重构切断撤销回写链路。

## 用户体验

用户在群聊任务卡里执行安全撤销后，全局主 Agent 会话也能看到清晰结论：

> 你派发到群聊主 Agent 的任务已安全撤销。

如果用户继续这个需求，主 Agent 会重新读取当前代码状态、重新规划、重新派发和重新验收，而不是复用已撤销的完成结论。
