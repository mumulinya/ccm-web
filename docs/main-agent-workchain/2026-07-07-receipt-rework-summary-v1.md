# 子 Agent 回执复检与返工小结 v1

## 背景

主 Agent 已经能评分子 Agent 回执，也能生成“精准返工建议”。但用户在任务卡上看到的是分散的协作状态，缺少一个直接回答“哪些子 Agent 的回执还不能验收、为什么、下一步怎么补”的小结。

## 本次升级

- 新增 `receipt_rework_summary`，从回执评分、缺失回执、异常 Worker 通知和已有精准返工建议中汇总回执缺口。
- 群聊任务卡新增“回执复检”区块，用户可以看到目标 Agent、缺口原因，并直接触发单项精准返工。
- 全局任务卡归一化逻辑兼容同一结构，跨项目任务如果存在弱回执，也能用相同展示。
- 原始 `CCM_AGENT_RECEIPT`、trace/session、完整返工指令仍放在技术详情或接口数据里，用户主视图只看业务可懂的小结。

## 用户可见效果

- 缺失回执：显示“要求子 Agent 补回执”。
- 弱回执：显示“要求补充高质量回执”，并列出缺少的 ACK、文件、验证、契约或记忆声明。
- 主 Agent 不会把弱回执任务直接判定完成，而是引导精准返工后重新验收。

## 验证

- 后端 UX 自测覆盖 `receipt_rework_summary` 和可触发的 `targeted_rework` 动作。
- 静态 UI 自测覆盖任务卡渲染、全局归一化和后端输出。
- Playwright 渲染回归覆盖真实页面里的“回执复检”和“返工”按钮。
