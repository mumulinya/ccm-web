# 用户可见术语优化 v1

## 背景

长期目标要求主 Agent 的回复像 Claude Code 一样，用户可见内容要友好、可读、能接上任务上下文。`D:\claude-code\src\constants\prompts.ts` 明确强调：用户回来时不应该被内部代号、缩写或过程术语增加理解负担。

此前任务卡已经把 trace、session、runtime 等技术字段默认折进“技术详情”，但部分可见区域仍有“回执”“接口/契约”“精准返工”“门禁通过”等偏内部协作词。真实用户只需要知道子 Agent 提交了什么结果、哪里还要补充、主 Agent 下一步会怎么验收。

## 本次升级

- 任务卡协作状态里的“回执检查”改为“结果说明检查”。
- “回执质量”改为“结果说明质量”。
- “回执复检”改为“结果复检”。
- “要求补充高质量回执”改为“要求补充高质量结果说明”。
- “接口/契约传递”改为“任务交接同步”，“接口/契约同步”改为“接口约定同步”。
- “精准返工”改为“定向补充”。
- 工具摘要里的“回执 N 条”改为“结果说明 N 条”。
- 用户可见的技术详情提示统一为“底层执行记录和排障信息默认收在技术详情里”。
- 回放 fixture 和截图断言同步到新文案。

## 用户可见规则

- 用户默认看到的是“结果说明、定向补充、结果复检”等自然语言。
- ACK、gate、receipt、trace、session、runtime 仍可保留在技术详情或子 Agent 内部工作单里。
- 普通问话仍不展示 Todo、任务卡或协作术语。

## 验收点

- 任务卡截图回归不再断言“回执复检/高质量回执/精准返工”。
- Replay 回归展示“结果说明 1 条”。
- 源码自测覆盖新的任务卡文案。
