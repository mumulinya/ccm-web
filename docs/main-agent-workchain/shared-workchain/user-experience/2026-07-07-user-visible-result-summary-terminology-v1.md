# 用户可见结果说明术语收敛 v1

本轮目标：继续对齐 `D:\claude-code` 的展示习惯。Claude Code 会把任务计划、执行、结果和总结展示给用户，但不会把内部协议名当成主要文案。本项目此前仍有一些默认可见区域写着“回执”，容易让普通用户误以为需要理解内部 Agent 协议。

## 改动

- 主 Agent 决策卡：
  - “读取子 Agent 回执”改为“读取子 Agent 结果说明”。
  - “等待子 Agent 执行和回执”改为“等待子 Agent 执行并提交结果说明”。
- Agent 协作看板：
  - 统计项、时间线、子 Agent 状态从“回执”改为“结果说明”。
  - 空状态改为“暂未收到子 Agent 通知或结果说明”。
- 任务中心：
  - 概览卡和验收证据里的“回执”改为“结果说明”。
  - 继续执行草稿改为“子 Agent 结果说明”。
  - 折叠技术详情标题改为“原始结果说明 / 结构化结果说明”。
- 斜杠菜单、系统诊断、记忆中心、定时任务默认模板、项目交付规范占位符和设置页说明统一使用“结果说明/执行结果通知”。
- 保留内部字段名和协议要求，`receipt` 数据结构与 `CCM_AGENT_RECEIPT` 契约仍用于主 Agent 验收和第三方写代码 Agent 交付。

## 验证

- `scripts/main-agent-decision-ui-selftest.mjs` 新增 `userVisibleReceiptTerminologyPolished`：
  - 覆盖主 Agent 决策卡、协作看板、任务中心、群聊状态、斜杠菜单、记忆中心、定时任务、项目工具、诊断和设置页。
  - 这些默认可见文件不再包含中文“回执”。
- 后续继续跑类型检查、构建、聊天体验和渲染回归，确认文案变化不影响任务卡与技术详情。
