# 回执返工复检结果闭环 v1

## 背景

上一版已经能把弱回执、缺失回执整理成“回执复检”缺口，并允许用户触发精准返工。但如果子 Agent 后续补齐了证据，任务卡上的缺口会消失，用户不容易知道“刚才那次回执返工已经被主 Agent 复检通过”。

## 本次升级

- `receipt_rework_summary` 新增 `active_rework` 和 `resolved` 两类状态。
- 当回执返工已经发起但目标 Agent 仍有缺口时，任务卡显示“等待回执”。
- 当目标 Agent 后续回执评分通过或不再存在回执缺口时，任务卡显示“回执已补齐 / 已复检”。
- 全局任务卡归一化逻辑同步支持已复检状态，跨项目任务也能保留返工闭环上下文。

## 用户可见效果

- 用户不仅能看到“需要返工”，还能看到“返工已发起”和“返工后已复检”。
- 主 Agent 完成复检后，会继续提示下一步是最终验收和总结。
- 技术字段仍留在技术详情，普通视图只显示可理解的业务状态。

## 验证

- 后端 UX 自测新增 `receiptReworkResolvedVisible`，验证回执补齐后仍显示已复检状态。
- 统一任务体验自测新增全局任务 `globalMissionShowsReceiptReworkResolved`。
- Playwright 渲染回归新增真实页面断言：`回执已补齐` 和 `已复检` 可见。
