# 主 Agent 完整工作链阶段验收 v1（已纠正）

> 纠正说明：本记录最初误标为“最终验收”。其中真实服务 E2E 只覆盖状态查询、SSE 顺序和重复请求回放，没有执行第三方代码 Agent、TestAgent、返工复验与最终总结，因此不能作为完整工作链最终通过证据。

## 目标验收范围

本轮目标覆盖群聊主 Agent 与全局主 Agent 从接收请求到最终交付的完整链路：

1. 普通问话直接友好回答，不展示 Todo。
2. 可执行需求先澄清范围，生成用户可见计划与实时 Todo。
3. 主 Agent 按项目或群成员派发执行，保留 Trace、任务和原生会话连续性。
4. 子 Agent 回传代码变更、验证和阻塞，主 Agent统一整理为用户可读进展。
5. TestAgent 独立复核；失败时定向返工、重新复核和主 Agent 抽查。
6. 只有强验收通过后才进入完成态，并给用户最终总结。
7. 等待补充、页面刷新、服务重启、重复消息和网络断线都继续原任务，不重复派发。

## 最终补齐项

- 群聊等待补充精确接回原任务。
- 群聊执行前澄清精确接回原请求。
- 全局澄清显式绑定原运行，刷新后恢复回答模式。
- 等待用户的全局监督器重启后不反复推进。
- 全局和群聊 SSE 具备事件 ID、顺序号与前端去重。
- 全局 `request_id` 和群聊 `client_message_id` 在断线重试时保持稳定。
- 全局请求失败立即释放实际持有的幂等租约。
- 用户主文本保持产品化表达，协议、会话 ID、Trace 和原始工具输出默认进入技术详情。

## 真实服务 E2E 覆盖

`scripts/main-agent-runtime-e2e.mjs` 使用隔离主目录启动当前生产构建，避免触碰用户现有任务与会话，并验证：

- 生产前端可访问。
- 全局状态请求通过真实 SSE 返回。
- 同一全局请求 ID 重发只回放原运行。
- 群聊主 Agent 状态请求通过真实 SSE 返回。
- 两条流的事件 ID 唯一且顺序严格递增。
- 用户可见主文本不暴露内部协议字段。

## 尚未通过的最终门禁

- 群聊主 Agent 在隔离项目中真实派发第三方代码 Agent 并产生代码修改。
- 全局主 Agent 在隔离项目中真实派发第三方代码 Agent 并产生代码修改。
- 两条链路均真实调用 TestAgent，并消费其结构化计划、证据和结论。
- 注入一次 TestAgent 失败，真实触发定向返工、复验和完成门禁。
- 主 Agent 在 TestAgent 后完成独立抽查，并输出用户可读最终总结。
- 执行中的真实任务经过进程重启、SSE 断线重发后仍保持同一任务且不重复副作用。

在以上门禁全部通过前，本文件只代表阶段验收，不代表长期目标完成。

## 维护约束

- 后续 TestAgent 内部实现仍由 `backend/test-agent/**` 的维护链路独立演进。
- 主 Agent 只消费 TestAgent 的结构化计划、证据和结论，不复制其内部业务流程。
- 新增主 Agent 功能必须继续更新本目录文档，并至少通过源码自测、类型检查和对应渲染或运行验收。
