# 回执复检重复信号去重 v1

## 背景

群聊主 Agent 的任务卡会同时接收完整 `receipts`、轻量 `receipt_statuses`、验收门禁和协调层返工建议。真实运行中，同一个子 Agent 可能已经提交了高质量完整回执，但历史轻量状态或泛化门禁仍残留“回执质量不足”的旧信号，导致用户看到已经补齐的回执仍被标成需要返工。

## 本次升级

- `receipt_rework_summary` 先按子 Agent 目标识别高质量完成回执。
- 如果同一目标已有高质量回执，不再展示重复的弱回执、缺失回执、ACK 或验证返工缺口。
- 如果只有无目标的泛化“回执质量”缺口，而已有高质量回执覆盖，也将其视为过期展示信号，不再打扰用户。
- 已发起过回执返工且目标缺口消失时，继续保留“回执已补齐 / 已复检”的可见结果。

## 用户可见效果

- 任务卡不会一边显示“回执已补齐”，一边又显示过期的“要求补充高质量回执”。
- 用户看到的是当前真实状态：需要返工、等待回执、或已复检通过。
- 技术来源、旧状态和原始协调信号仍保留在技术详情里，普通视图只呈现可执行结论。

## 验证

- 打包产物 `runCollaborationUxSelfTest()` 通过 `receiptReworkResolvedVisible`。
- Playwright 渲染回归验证 `回执已补齐`、`已复检` 和返工缺口不会互相冲突。
