# 主 Agent 完成前收尾摘要 v1

## 目标

群聊主 Agent 原本已经有真实的完成闸门：执行队列未收敛、子 Agent 会话未结束时，任务不能标记完成。本次不重复实现判定逻辑，而是把闸门原因转换成用户看得懂的“完成前收尾”，让用户明确知道为什么还不能宣布完成。

## 本次实现

- 新增 `ccm-main-agent-completion-readiness-v1` 用户可见摘要。
- 群聊任务卡和全局任务卡统一透传并展示收尾状态。
- 未完成时显示工作项数量、仍在处理的子 Agent 会话数量和下一步。
- 工作项行只展示 Agent、工作内容和友好状态，不展示内部工作项 ID。
- 主 Agent 验收区新增“执行队列收尾”和“子 Agent 会话收尾”两项检查。
- 后端原始闸门信息、工作项 ID、会话 ID 继续保留在默认折叠的技术详情中。
- 普通问话没有任务卡和工作项，不展示 Todo 或完成前收尾。

## 用户可见行为

- 有未完成工作时显示“尚未收尾”，并说明剩余工作项和会话数量。
- 工作项会显示“执行中”“等待开始”“等待处理”等友好状态。
- 队列和会话全部结束后显示“可以总结”，主 Agent再进入最终验收和总结。
- 主 Agent 不会因为部分子 Agent 已返回结果就提前向用户宣布完成。

## 回归覆盖

- 后端协作 UX 自测覆盖友好阻塞说明、验收检查名称和技术 ID 隔离。
- 静态 UI 自测覆盖群聊/全局字段透传和组件渲染。
- Playwright 渲染回归覆盖群聊与全局任务卡：
  - “完成前收尾”和阻塞状态真实可见。
  - 工作项内容与友好状态真实可见。
  - 工作项 ID、会话 ID 不出现在用户可见区域。
  - 分别生成群聊和全局“完成前收尾”元素截图，便于直接检查排版。
