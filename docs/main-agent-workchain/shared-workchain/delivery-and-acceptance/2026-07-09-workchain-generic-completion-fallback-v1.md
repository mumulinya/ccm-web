# Workchain 通用完成兜底收紧 v1

## 背景

主 agent 的 workchain 是群聊主 agent、全局主 agent 共同使用的最终总结结构。旧逻辑在 `status=completed` 且没有可展示证据时，会兜底展示“已完成本轮处理，并整理了结果给你”。

这类文案对普通回复还可以接受，但对开发/派发任务容易造成“没有验收证据也显示完成”的误解。

## 本次调整

- 普通对话 completed 兜底改为“回复已整理给你”。
- 任务/派发类 completed 但无证据时，兜底改为“处理结果已整理，是否已经交付以验收和最终总结为准”。
- 格式化完成总结中“我已完成本轮处理，但没有更多可展示的业务证据”改为“处理结果已整理，但没有更多可展示的业务证据”。
- `runMainAgentWorkchainSelfTest` 新增 `workchainGenericCompletionFallbackAvoidsFalseDone`。
- 总静态自检锁定新文案，并禁止旧无证据完成兜底回归。

## 验收口径

有文件、验证、独立复核、执行目标完成等证据时，仍可以展示“已完成：...”。缺少证据时，只能展示结果已整理，并引导用户看验收和最终总结。
