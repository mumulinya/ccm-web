# 前置派发阶段完成态措辞收紧 v1

## 背景

主 agent 链路已经有计划、执行、验收和总结，但前置阶段里仍有一些用户可见文案使用“已完成”：

- 执行前计划已完成；
- 只读检查已完成；
- 我已完成派发；
- 需求分解完成。

这些动作只是进入执行链路前的准备或派发，不等于用户需求已经验收完成。参考 CC 的任务链路原则，前置计划和派发应表达为“已就绪 / 已发出 / 已整理”，最终“完成”只留给验收后的交付总结。

## 改动

- 群聊主 agent 的执行前计划自动继续文案改为“执行前计划已就绪”。
- 执行前检查文案改为“只读检查已通过”。
- 派发判断 fallback 改为“派发判断已整理”。
- 需求分解提示改为“需求已分解”。
- 全局主 agent 派发进展 fallback 改为“派发已发出，正在跟踪后续结果”。

## 用户体验

用户看到的是：

- 计划已经准备好；
- 派发已经进入任务链路；
- 后续还要等待执行成员结果、验收和最终总结。

这样不会把“前置计划完成”误读成“需求完成”。

## 验证

- 静态自检新增 `preCompletionDispatchCopyAvoidsFalseDone`。
- 该自检要求后端前置阶段不再包含旧的“执行前计划已完成 / 已完成派发 / 需求分解完成”等完成态措辞。
