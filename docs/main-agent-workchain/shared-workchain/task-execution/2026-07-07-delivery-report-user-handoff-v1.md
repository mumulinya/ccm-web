# 交付报告携带用户收尾建议 v1

## 背景

参考 `D:\claude-code` 的主 Agent 链路，子 Agent 的原始结果不应该直接变成用户主要可见内容，主 Agent 需要把执行结果整理成用户能看懂的总结、下一步和风险说明。

此前前端任务卡可以根据执行结果推导 `ccm-main-agent-user-handoff-v1`，但统一交付报告本体没有稳定携带这个结构。这样在全局历史、非前端消费方、最终回复归档中，可能只能看到完成内容，缺少明确的“接下来建议”。

## 本次升级

- 后端 `ccm-main-agent-delivery-report-v1` 增加 `user_handoff` / `userHandoff`。
- 交付报告 `sections` 增加“接下来建议”，与完成内容、涉及范围、验证结果、风险一起进入统一报告。
- 全局任务和群聊任务卡优先使用后端交付报告里的 `user_handoff`，避免前端重复推导覆盖服务端最终判断。
- 视觉回归 fixture 让 `delivery_report` 本体携带 `user_handoff`，截图断言同时覆盖任务卡展示和交付报告区块。
- 自测补充“只有 `final_delivery_report.user_handoff` 时也能展示”的路径。

## 用户可见规则

- 普通问话仍不展示 Todo 和任务收尾建议。
- 用户可见区域只展示友好的完成状态、建议动作、证据和待处理项。
- Trace、会话、执行器细节、原始回执等继续默认收在技术详情里。

## 验收点

- 完成任务时，交付报告和任务卡都能展示“接下来建议”。
- 有风险时，主动作优先提示处理风险与待确认项。
- 失败时，主动作是重新执行或继续修复。
- 取消时，提示任务已停止，主动作是重新发起需求；停止原因只作为辅助信息展示。
- 全局历史同步后，结构化交付报告不会退化成纯文本。
