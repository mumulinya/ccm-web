# Plan Approval Request Summary V1

日期：2026-07-08

## 背景

对照 `D:\claude-code` 的 `ExitPlanMode` 链路，计划审批不只是展示计划，还会明确告诉用户和模型：批准后可以开始执行、被编辑的计划会成为执行依据、拒绝时要继续调整计划。

本项目已经有执行前计划、确认按钮、补充要求输入框和确认后的执行跟进，但等待确认阶段还缺少一段稳定的用户可见说明，告诉用户“确认后会解锁什么、确认前不会做什么、补充要求会如何进入执行链路”。

## 本次升级

- 在群聊/全局共用的任务体验卡中新增 `ccm-main-agent-plan-approval-request-v1` 派生摘要。
- 仅在计划确实等待确认时显示；已确认、自动继续或任务终态不会显示。
- 摘要会说明：
  - 确认后主 Agent 才会开始执行。
  - 确认前不会做写入、派发或高风险操作。
  - 确认后会继续的步骤。
  - 执行边界、证据跟踪和最终验收总结规则。
  - 用户可以在确认前补充要求，补充内容会带入执行计划。
- 群聊主 Agent 的执行前确认卡和全局主 Agent 的实时计划卡都会显示该摘要。

## 用户可见边界

用户主视图显示的是确认说明、执行边界和下一步，不展示原始 trace、底层 payload、技术事件或权限协议字段。

## 验证

- 静态自测新增 `taskCardRendersPlanApprovalRequest`。
- Playwright 渲染回归覆盖：
  - 群聊执行前确认卡显示“等待你确认计划”。
  - 全局实时计划卡显示“等待你确认计划”。
  - 自动继续的全局计划卡不显示确认请求摘要。
