# 交付报告计划回顾 v1

## 背景

长期目标要求群聊主 Agent 和全局主 Agent 能形成清晰的计划、执行、验收和总结。此前任务卡能展示 Todo、执行前计划和计划执行核对，但统一交付报告的纯文本总结主要围绕完成内容、验证、验收和风险。

这意味着用户只看最终回复、全局历史或非前端消费方时，可能知道“做完了什么”，却看不到“原来准备怎么做，以及最后是否按计划对齐”。参考 `D:\claude-code` 中 TodoWrite 和 final summary 的思路，计划应帮助用户跟踪进展，但最终总结也需要用用户能懂的话交代计划是否闭环。

## 本次升级

- `ccm-main-agent-delivery-report-v1` 增加“计划回顾”section。
- 后端新增计划回顾汇总逻辑，从 `plan_mode`、Todo steps、验收标准和 `plan_alignment` 中抽取用户可读内容。
- 交付报告增加 `plan_review` / `planReview` 字段，供全局历史、群聊任务卡和非前端消费方复用。
- Pickup 摘要增加“计划：...”复看项，用户回来继续时先看到原计划。
- Pickup 摘要保持风险/停止原因优先，计划回顾只补充上下文，避免把需要用户处理的事项挤出复看入口。
- 用户收尾建议的证据摘要增加计划信息。
- 前端交付报告展示上限从 7 个 section 扩到 8 个 section，确保“计划回顾”和“接下来建议”都能展示。

## 用户可见规则

- 有计划时展示执行前计划、计划范围、计划步骤、验收标准和计划核对状态。
- 无单独计划记录时，只给温和兜底：“主 Agent 已按交付证据整理结果”。
- 不展示 Todo 原始 JSON、trace、session、runtime 或子 Agent 原始回执。
- 普通问话仍不展示 Todo、任务卡或交付报告。

## 验收点

- 统一交付报告 markdown 出现“计划回顾”。
- 计划回顾能显示计划步骤和“计划核对：已对齐”。
- 任务卡交付报告 grid 同时显示“计划回顾”“验收结论”“接下来建议”。
- 全局历史保留 `plan_review`。
- 构建产物中的后端自测覆盖 `groupHasPlanReview`。
