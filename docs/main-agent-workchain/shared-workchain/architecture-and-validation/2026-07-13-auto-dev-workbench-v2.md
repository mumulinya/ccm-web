# 自动开发工作台 V2

## 目标

把自动开发页面从“日报、飞书、Cron 配置混排页”调整为面向用户的完整工作台：用户先看到能否运行、当前进展和阻塞，再按需查看报告或设置通知。技术诊断保留为折叠详情或独立管理入口。

## 页面边界

- 自动开发页负责：运行状态、立即运行、今日进展、阻塞续跑、需求池摘要、日报周报和飞书报告通知。
- 定时任务页负责：创建、编辑、启停、Cron 表达式、立即触发和运行记录。
- 自动开发页只显示“无人值守已启用多少个计划”，并提供到定时任务页的管理入口。
- 系统设置继续负责底层模型、飞书通道和执行环境配置，不在自动开发首页铺开技术参数。

## 用户工作流

1. 页面读取 `/api/auto-dev/overview`，展示当天任务、需求池、完成和阻塞数据。
2. 页面读取 `/api/orchestrator/diagnostics`，把群聊、项目子 Agent、验收和队列状态转成用户可读的运行准备结果。
3. 用户点击“立即运行”后，调用 `/api/orchestrator/daily-dev-autopilot/run`。
4. 后端优先续跑交付缺口，再整理共享资料、认领可接活需求，并把任务交给群聊主 Agent 和项目子 Agent。
5. 页面展示本轮续跑、资料整理、派发和入队数量，并每 15 秒刷新今日进展。
6. 日报和周报使用结构化组件渲染；命令、CLI 错误等原始内容默认收进“技术详情”。

## 三个视图

### 运行概览

- 自动开发可用状态和友好原因
- 执行中、待处理、今日完成、需要关注
- 今日开发活动和阻塞续跑
- 自动开发链路准备项
- 无人值守与需求池摘要

### 报告

- 日报和周报分段切换
- 历史日期选择
- 重新生成和发送到飞书
- 安全的结构化 Markdown 解析，不使用 `v-html`
- 原始技术内容默认折叠

### 通知设置

- 日报和周报独立开关
- 周期和发送时间
- 手动发送
- 最近通知结果

## 响应式修复

- 页面改为单一纵向滚动容器，移除原来的主栏、侧栏双滚动。
- 手机端不再让主报告区和通知/调度区占用同一网格行。
- 手机端统计改成两列，表单改成单列，底部保留导航安全空间。
- 切换视图时回到页面顶部，避免从长报告切换后落在通知页中段。
- 周报复选框保持固定尺寸，不再被通用输入框宽度规则撑开。

## 自动化验证

- `npm run test:auto-dev`：页面结构、真实执行接口、Cron 边界、报告渲染和导航源码自测。
- `npm run test:auto-dev-api`：概览、自动开发诊断、飞书配置和后端路由契约审计。
- `npm run test:auto-dev-render`：桌面和手机真实浏览器渲染，断言无横向溢出、无面板重叠、技术详情默认折叠、复选框尺寸稳定和视图切换回顶。
- `npm run check`：后端与飞书集成 TypeScript 检查。
- `npm run build`：前端、后端和飞书集成完整构建。
- `npm run test:runtime-paths`：运行路径引用完整性检查。
