# 证据驱动的工作日报与周报 V1

日期：2026-07-13

## 目标

日报和周报用于回答“我今天或本周实际做了什么”，不再根据任务当前状态回算历史，也不让模型脱离事实自由补写。用户默认看到易读复盘，事件 ID、Schema 和原始来源统计只在技术详情或审计 API 中提供。

## 工作事件账本

- 持久化文件：`~/.cc-connect/work-journal.jsonl`
- 事件 Schema：`ccm-work-journal-event-v1`
- 报告 Schema：`ccm-evidence-work-report-v2`
- 写入方式：稳定事件 ID 去重，只追加，不修改已有事件。
- 清理策略：默认不自动清理，审计接口明确返回 `append_only=true` 和 `auto_cleanup=false`。
- 时间口径：按本机时区的自然日；周报按周一 00:00 到下周一 00:00 汇总。

每条事件记录时间、事件类型、状态、工作归属、来源、任务/群聊/项目关联、证据等级、证据引用和必要元数据。

## 已接入来源

- 任务创建、排队、执行、完成、阻塞和工作流时间线。
- 工作台执行前确认任务；创建动作归属用户，后续执行按实际事件归属。
- 全局助手会话、群聊全部会话和项目会话。
- 飞书进入的任务会话与飞书来源标记。
- 代码文件变更、构建/测试/验证记录和交付摘要。
- 任务内 TestAgent 复核，以及能匹配真实任务 ID 的 TestAgent runner 记录。
- 定时任务和自动开发最近运行记录。

没有真实任务关联的 TestAgent 硬化自测不会进入用户日报，避免把系统自测写成用户工作。

## 归属规则

- `user`：用户在全局助手、群聊、项目会话、飞书或工作台发起的真实工作需求。
- `agent`：全局 Agent、群聊主 Agent、项目子 Agent 的计划、派发、执行、文件变更、验证和交付。
- `test_agent`：TestAgent 或独立复核形成的验收证据。
- `system`：队列状态变化、定时任务和自动开发运行。

报告按 `actor_type` 分类，不再根据显示名称猜测归属。一个工作项包含多种归属时可在对应段落中分别出现，并在活动行保留参与者摘要。

## 日报与周报口径

日报包含：今日完成、用户发起或推进、Agent 推进、系统自动化、验收与质量、仍在进行、风险与待处理、明日继续和数据依据。

周报包含：本周完成、重点推进、代码与交付、验收与质量、风险与未完成、下周继续和本周工作依据。

“仍在进行”由报告日结束时的事件状态推导。周一创建、周五完成的任务在周一日报中仍是待处理，不会因为当前已完成而回写成周一完成。过去日期或已结束周的 V2 快照默认不重写；只有显式 `force=true` 才允许重建。

明日和下周计划只来自当前阻塞项、仍在执行项和可接活需求，不使用固定套话填充。

## API

- `GET /api/auto-dev/overview`：返回当前复盘、历史日报/周报和轻量账本摘要，不返回完整事件数组。
- `GET /api/auto-dev/work-journal/audit`：返回账本总量、来源/归属统计、时间范围和契约自测结果。
- `GET /api/auto-dev/work-journal/events`：支持 `start`、`end`、`task_id`、`source`、`limit` 条件查询，最多返回 1000 条。
- `POST /api/auto-dev/report/generate`：生成日报，可显式传入 `date` 和 `force`。
- `POST /api/auto-dev/weekly-report/generate`：生成周报，可显式传入 `date` 和 `force`。

## 用户界面

自动开发页的“报告”视图更名为“工作复盘”。默认展示：

- 工作事件数、强证据数和数据来源数。
- 用户、Agent、TestAgent 与系统自动化的归属统计。
- 结构化、可换行的日报或周报正文。
- 默认折叠的技术详情，仅包含 Schema、生成时间、不可变来源标记、证据统计和事件 ID。

桌面与手机均限制长任务标题的固有宽度，活动行不会撑破页面。

## 当前边界

- 账本只能投影现有持久化事实；早于当前系统记录保留期、且已被清理的原始事件无法恢复。
- 对话只有满足工作意图、任务关联或附件条件时才进入复盘，普通问候和普通问答不会被当成工作。
- TestAgent runner 必须能绑定真实任务 ID；无法绑定的自测仅保留在 TestAgent 自身审计中。
- 自动开发执行链路是否可运行由独立诊断接口决定，报告可正常生成不代表所有项目子 Agent 当前均可执行。
