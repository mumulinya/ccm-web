# 子 Agent 执行计划复核 v1

日期：2026-07-07

## 背景

Claude Code 的团队链路里有 `PlanApprovalMessage`：队友提交计划后，负责人可以看到计划、批准或退回。CCM 的群聊主 Agent 已有子 Agent 工作单、ACK 前置审核和结果说明检查，但用户侧只能看到零散的“ACK 缺少目标或计划范围”，不够像一个完整的“计划已提交 / 主 Agent 已复核 / 可继续执行”链路。

## 本次升级

- 后端 `agent_coordination` 新增 `ccm-child-agent-plan-review-v1`。
- 主 Agent 会把子 Agent 接单 ACK 里的目标、计划范围、验证安排和待澄清项整理成用户可读摘要。
- 前端任务卡新增“子 Agent 执行计划”区块：
  - 已通过：显示子 Agent 的目标、范围和验证计划。
  - 需调整：显示哪些子 Agent 的计划还不够清楚，以及需要补齐什么。
  - 等待提交：显示主 Agent 正在等待子 Agent 提交接单计划。
- 历史任务如果还没有 `child_plan_review`，前端会从已有 `ack_review` 兜底生成同样的用户视图。
- `ACK`、Trace、原始协议仍不直接暴露在用户正文里；用户看到的是“接单确认/执行计划/验证安排”。

## 回归覆盖

- 后端自测覆盖通过态和需调整态。
- 静态自测覆盖后端生成、前端渲染、fixture 与 Playwright 断言。
- Playwright 渲染回归检查：
  - 完成任务卡展示“子 Agent 执行计划”和验证计划。
  - 工作队列卡展示“需调整”状态和待澄清项。
  - 用户可见区域不显示底层 `ACK` 文案。

## 用户体验原则

用户应该理解“主 Agent 不是盲目派发，而是在看子 Agent 是否真的理解目标、范围和验证方式”。技术协议可以存在，但默认收在技术详情或内部结构里。
