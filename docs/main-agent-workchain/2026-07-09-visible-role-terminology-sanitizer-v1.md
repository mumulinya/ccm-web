# 用户可见角色文案清洗 v1

日期：2026-07-09

## 背景

主 Agent 和群聊/全局协调链路里，内部数据仍会使用 `主 Agent`、`子 Agent`、`群聊主 Agent`、`全局主 Agent` 等技术角色名。但这些词直接出现在默认聊天文本、Todo 当前步骤、进度刷新和派发卡片里时，用户会看到实现细节，不够像 Claude Code 那样以“我正在做什么、下一步是什么”的方式表达。

## 改动

- 在前端公共展示清洗层中补齐角色术语转换：默认展示把主协调者写成“我”，把子执行者写成“执行成员”，把群聊主协调者写成“协作群”。
- 决策卡的 Todo 步骤、当前焦点、下一步和处理链路先清洗再渲染，避免 `todo_plan.steps` 的原始字段直接露出。
- 群聊状态卡的当前步骤、最近进展、执行成员状态和最近决策预览统一走友好展示文案。
- 全局流式消息继续保留技术详情中的原始结构，但默认事件和进度卡会展示清洗后的用户语言。

## 回归保护

- 截图回归新增断言：任务计划卡、群聊当前步骤卡、全局流式派发卡默认可见区域不显示内部角色名。
- 静态自测新增守卫：公共 sanitizer、决策卡、群聊状态卡、最近决策预览和渲染回归断言必须同时存在。

## 预期体验

用户看到的是“我会继续验收并总结”“等待执行成员提交结果说明”“协作群已收到安排”等业务化文案；内部协议、角色名和原始执行结构仍保留在技术详情中，便于排障和审计。
