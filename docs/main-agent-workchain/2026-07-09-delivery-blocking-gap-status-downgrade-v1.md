# Delivery Blocking Gap Status Downgrade v1

## 背景

上一轮已经让独立复核未通过时的下一步指向返工和重新复核。但如果交付报告顶部仍显示“状态：已完成”，用户还是可能误判任务已经交付完成。CC 风格要求复核、验证或验收失败时必须如实报告，不能把失败包装成完成。

## 本次修改

- 新增阻塞性完成缺口判定。
- 当底层状态是 `done`，但存在以下硬缺口时，交付报告的用户可见状态会降级为“未完成”：
  - 验收门禁未通过。
  - 独立复核必需但未通过。
  - 必需验证缺失或验证来源门禁失败。
  - 存在失败验证记录。
  - 计划核对仍有缺口。
- 普通的完成后提醒，例如上线前配置密钥，不会触发状态降级。

## 验证

- 运行时自测覆盖“状态为 done，但独立复核未通过”的场景。
- 自测确认主文本显示“状态：未完成”，并且不再显示“状态：已完成”。
- 静态自测覆盖阻塞缺口判定函数。
