# 群聊主 Agent 待确认问答文案

日期：2026-07-09

## 背景

群聊主 Agent 的状态卡和状态追问摘要里，待确认问答会显示为“开放问答”或“Agent 问答需要处理”。这些说法偏内部实现，不利于用户理解当前到底是否需要自己确认。

## 本次调整

- 前端群聊协作状态卡将“开放问答”改为“待确认问答”。
- 后端状态追问摘要将“Agent 问答需要处理”改为“待确认问答需要你确认”。
- 渲染 fixture 增加 `open_qa_count: 1` 的群聊当前状态场景。
- Playwright 回归断言新文案可见，并确认旧术语不再展示。
- 源码自测同时守住前端标签和后端摘要文案。

## 验收点

- 用户能直接看懂：当前有待确认问答，需要自己确认。
- 内部 “Agent 问答” 不再出现在用户可见状态摘要里。
- 普通任务进度、当前 Todo 和技术详情折叠逻辑保持不变。
