# Workchain Weak Acceptance Verification Gate v1

## 目标

参考 Claude Code 的交付口径：主 Agent 不能只因为内部字段标记“最终验收已通过”就把任务展示成完整完成。只要任务链路里有实际执行或文件变更，但缺少真实验证、复核或验收明细，就必须继续提醒补齐证据。

## 实现

- 新增工作链路层的强验证证据判断：
  - 实际验证记录；
  - 独立复核记录；
  - 带明细的验收项。
- 单独一句“最终验收已通过”不再算作强验证证据。
- 如果验证文本里出现失败、错误、非零退出码等信息，最终总结质量门禁会判定为未通过。
- 质量补齐 Todo 不会冒充原计划里的验证步骤，原有“缺少验证/验收步骤”提醒仍可保留。

## 用户体验

- 用户不会因为一个乐观验收标记看到“已完成”的假象。
- 主文本会说明最终交付总结还在补齐。
- 技术细节和底层字段仍默认放在技术详情里。

## 自测

- 新增 `workchainWeakAcceptanceOnlyBlocksFalseCompletion`：
  - 有文件变更；
  - `acceptance_gate_passed=true`；
  - 没有验证、复核或验收明细；
  - 预期最终总结质量不通过，并保留补齐 Todo。
