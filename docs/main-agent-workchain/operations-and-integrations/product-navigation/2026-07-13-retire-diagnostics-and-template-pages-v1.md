# 系统自检与对话模板页面下线 V1

## 目标

精简普通用户可见的导航与页面，不再将系统自检和对话模板作为独立工作台页面，同时保留主 Agent 与现有业务仍需要的底层能力。

## 改动

- 移除主导航中的“系统自检与体检”和“对话模板”。
- 取消两个独立页面的异步加载与渲染，构建产物不再生成对应页面分包。
- 删除已不可达的 `SystemDiagnostics.vue` 和 `Templates.vue`，同步移除源码自测对它们的旧依赖。
- 清理菜单分组与清理中心的旧页面入口。
- 旧 `diagnostics` 链接自动转到系统设置，旧 `templates` 链接自动转到工作台，避免书签或历史链接打开空白页。
- `/doctor` 改为直接读取 `/api/orchestrator/diagnostics`，不再跳转页面。
- 移除 `/templates` 导航命令与全局 Agent 的模板页面识别。

## 保留能力

- 保留 `/api/orchestrator/diagnostics`，设置页、任务页、定时任务和主 Agent 仍可使用真实诊断数据。
- 保留 `/api/templates` 与现有聊天输入框兼容链路，本次只下线独立管理页，不扩大为数据删除。

## 验收标准

- 导航和菜单管理不再出现两个页面。
- 清理中心不再跳转到已下线的系统自检页。
- 旧链接有稳定的替代落点，不会白屏。
- 前端构建和后端类型检查通过。

## 实际验证

- `npm run build:frontend` 通过，构建清单中已无 `Templates` 和 `SystemDiagnostics` 分包。
- `http://127.0.0.1:3080/` 返回 HTTP 200，并已加载本次新生成的主资源。
- 入口扫描通过：页面导入、Tab 声明、渲染节点、菜单分组、清理中心死链和模板页导航命令均已移除。
- 旧链接重定向和 `/doctor` 诊断 API 查询定义已通过源码断言。
- `npm run check` 复检通过，后端与飞书集成 TypeScript 类型检查均无错误。
- 内置浏览器当前无可用标签页，本次未能生成交互截图；已以生产构建、HTTP 服务和资源清单完成运行验证。
