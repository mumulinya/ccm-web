# 音乐 Agent 对话可靠性升级

日期：2026-07-13

## 问题

网易云模式调用统一大模型时返回 `Missing required parameter: input[n].content[0].text`。音乐页面在创建流式回复占位消息后，把这条空 Agent 消息连同当前问题一起作为历史发送；后端又重复追加当前问题。严格的 OpenAI Responses 兼容网关会拒绝空文本块。

## 调整

- 前端请求历史排除正在生成的空消息，只发送用户与 Agent 的非空纯文本。
- 后端统一规范角色、提取字符串或文本块、移除空消息并合并连续同角色消息。
- 后端删除历史中重复的当前问题，确保对话从用户消息开始。
- OpenAI 兼容和 Anthropic 请求在发送前执行同一套最终清洗。
- 模型错误改为用户可理解的提示，原始响应只写入服务日志。
- 修正系统提示中的当前音乐模式插值，确保模型明确使用网易云、本地或 B 站结果格式。
- 将模式、助手状态、加载状态和曲目操作改为清晰中文，聊天区域由“转码工作台”更名为“音乐助手”。
- 修复窄屏下曲库与助手占用同一网格行的问题；手机端优先展示音乐助手，并精简底部播放控制避免重叠。
- 增加音乐 Agent 历史格式回归测试，覆盖空占位消息、重复问题和结构化文本块。

## 验收

- `npm run test:music-agent`
- `npm run check`
- `npm run build`
- 正式服务网易云模式真实对话不再返回空文本参数错误。

真实服务验收结果：

- 使用原始故障结构（包含空 Agent 占位消息和重复当前问题）请求网易云模式，HTTP 200，收到 `music_action`、流式文本和 `done`，没有 `error` 事件。
- 搜索“周杰伦的晴天”识别为 `search_music`，真实网易云搜索和 `tracks` 推荐均正常返回。
- 页面切换到网易云后发送“你好”，显示正常中文回复，浏览器控制台无错误。
- 390px 手机视口无页面横向溢出，音乐助手与曲库没有网格重叠，底部播放器没有越界。
