# 任务回放逐行代码变更 v1

日期：2026-07-13

## 目标

让用户在完整任务回放中从“代码与命令记录”直接查看具体文件、修改前后代码和准确行号，而不再只看到文件名列表。

## 实现

- 回放接口保留任务执行时持久化的统一 diff，并为每个文件返回项目、状态、增删行数和可用性。
- 代码变更卡片展示逐行变更覆盖情况，并提供“查看具体代码变更”入口。
- 代码变更抽屉支持文件切换、搜索、单栏、左右对照和完整文件模式；单栏模式展示修改前、修改后双行号。
- 新生成的统一 diff 使用标准 hunk 行号，多个相隔改动不会因省略上下文而显示错误行号。
- 历史任务若只保存了文件名或增删统计，会明确说明无法还原逐行内容，不读取当前工作区差异冒充历史证据。
- diff 输出继续执行密钥、本机绝对路径脱敏和长度上限控制。

## 数据保留

逐行 diff 跟随任务及执行记录持久化，在任务被永久删除前可用于回放。旧任务是否能逐行查看取决于当时是否保存了原始 diff。

## 验证

- 后端契约自测覆盖原始 diff 保留和旧记录不可还原提示。
- Playwright 渲染回归会点击代码变更卡片，断言文件、删除行、新增行以及修改前后行号可见，并保存真实截图。
