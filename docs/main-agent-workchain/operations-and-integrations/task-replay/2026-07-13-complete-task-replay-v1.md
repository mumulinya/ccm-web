# 完整任务回放 v1

日期：2026-07-13

## 目标

让用户按任务查看并排查完整业务链，而不是只看到零散 Trace：

`用户 / 全局主 Agent -> 群聊主 Agent -> 项目子 Agent -> 群聊验收 -> TestAgent -> 返工与复验 -> 完成前抽查 -> 最终总结`

普通问话仍按普通消息展示，不创建或展示任务回放卡。只有已经持久化为任务的工作才进入回放索引。

## 实现

### 后端聚合

- 新增 `backend/modules/collaboration/task-replay.ts`。
- 以任务 ID 为入口，自动向上寻找父任务，再收集全部子任务与同一 global mission 的任务。
- 关联任务时间线、任务日志、群聊消息、执行内核、项目 Agent 会话、可靠性 Trace、全局 Agent run、mission supervisor、TestAgent runner 与 TestAgent 证据清单。
- 新增按任务追加的 `reliability/task-replay-journal/*.jsonl` 完整日志。所有带任务 ID 的新 Trace 事件同时写入该日志，不受快速 Trace 的 1200 条滚动上限影响。
- 所有来源归一为按时间排序的事件，包含阶段、状态、参与者、任务、项目、摘要、技术详情和证据 ID。
- 用户时间线默认隐藏通过状态的底层 Trace；用户可打开“底层事件”查看聚合到的全部轨迹。异常 Trace 始终显示。
- 老 Trace 无法关联任务时仍可打开兼容视图，并明确提示其完整度有限。

### API

- `GET /api/tasks/replay?limit=100`：最近可回放根任务索引。
- `GET /api/tasks/replay?task_id=<id>`：父子任务完整回放。
- `GET /api/tasks/replay/artifact?task_id=<id>&run_id=<id>&artifact_id=<id>`：读取经过任务归属校验的 TestAgent 原始证据。
- `GET /api/tasks/replay/self-test`：回放契约与脱敏自测。

### TestAgent 证据

- `backend/test-agent/artifact-retention.ts` 新增按任务读取的公开证据目录。
- 目录只返回哈希证据 ID、类型、标题、大小、预览类型、完整性摘要和保留状态，不返回本机路径。
- 支持截图、结构化报告、文本日志、HAR、视频和其他下载证据。
- 相对路径和绝对路径都必须经 `realpath` 解析，并且最终文件必须位于对应 TestAgent run 目录内。
- 证据读取 API 不接受文件路径，因此路径穿越、跨任务读取和任意文件读取都会返回 404。

### 前端

- `TraceReplay.vue` 从 Trace 工具升级为任务回放页。
- 用户界面不再显示 Trace ID 输入框或以 Trace ID 命名旧记录；Trace 仅保留为后台关联、可靠性诊断和旧数据兼容能力。
- 独立“任务回放”侧栏入口已移除，用户从群聊/全局任务卡进入；回放标签页与最近任务列表仍保留。
- 技术详情、任务管理、工作台和系统诊断中的可见 `Trace / Trace ID / Trace Replay` 文案统一改为“执行记录”，且不展示原始追踪编号。
- 最近任务列表支持标题、目标和任务 ID 搜索。
- 回放页显示完整耗时、事件数、子任务数、TestAgent 运行数、问题数和证据数。
- 支持阶段、状态、参与者、父子任务、文本搜索及“问题 / TestAgent / 浏览器验证 / 改动与返工”快捷筛选。
- “上一项 / 下一项”可以直接定位失败、受阻和警告事件。
- 父任务和项目子任务可以单独筛选，但保持在同一时间轴中。
- 事件摘要保留换行；技术详情默认折叠。
- TestAgent 截图可直接预览，报告与日志可打开原始证据；已过期证据保留摘要并明确标记不可用。
- 群聊任务卡与全局任务卡都会携带 `task_id` 打开同一任务回放。

## 数据保留

- 任务、业务时间线和最终总结：任务永久删除前保留。
- 完整任务日志：任务永久删除前保留，不做条数截断。
- 快速可靠性 Trace：单 Trace 保留最近 1200 个事件，用于常规诊断；完整回放从任务日志补齐更早事件。
- TestAgent runner 记录：默认 14 天，最多 500 条。
- TestAgent 原始证据：默认 14 天，最多 200 次运行，总容量默认 2GB。
- 原始证据到期不会删除任务时间线摘要；回放会显示到期状态和策略。
- 永久删除任务时同步清除 TestAgent runner 与证据，运行中的任务使用 tombstone 防止完成回调恢复已删除记录。
- 永久删除任务时同步清除该任务的完整回放 JSONL；普通归档不会清除，恢复后仍可继续回放。

## 安全和可读性

- 用户内容会隐藏密钥、token、密码、Authorization 和内部回执协议。
- 技术详情会移除本机用户目录、工作目录、MCP 配置路径、原生第三方 Agent 会话 ID 和内部快照路径。
- 代码改动只显示项目相对路径；无法确认相对关系的绝对路径只显示文件名。
- 原始证据使用 `private, no-store` 和 `nosniff` 返回。
- 全局 Agent 仍不直接调用 TestAgent。TestAgent 事件来自群聊主 Agent 验收后的固定责任链。

## 已知边界

- 历史任务只能回放当时确实持久化过的数据；旧 Trace 若没有任务 ID 关联，只能展示兼容诊断视图。
- 2026-07-13 之前的历史任务没有完整 JSONL，仍受当时各数据源的滚动上限约束；升级后的新事件会完整追加保存。
- TestAgent 原始证据受保留期限和容量上限约束。面向用户的结论、失败原因和证据元数据不会随原始文件一起消失。
