# Windows Git 弹窗修复

## 问题

进入“代码协作”页面后，Windows 会反复显示标题为 `git.exe` 的控制台窗口。页面初始化需要连续读取分支、工作区状态、远端状态和首个文件 Diff，因此一次进入可能启动多条 Git 命令。

## 原因

代码协作后端通过 `execFileSync("git", ...)` 执行 Git，但没有设置 Windows 子进程隐藏参数。从浏览器或后台 CCM 服务触发时，每条命令都可能创建可见控制台窗口。

## 修复

- Git 执行器统一设置 `windowsHide: true`。
- 保留标准输入、输出和错误管道，错误处理和 Git 结果不变。
- 不使用 Shell 包装，不改变命令参数、仓库路径或确认门禁。
- 增加发布回归断言，防止后续重构移除隐藏窗口参数。

## 验证

- 后端 TypeScript production build。
- 代码协作真实项目状态、Diff、提交预检和远端安全门禁回归。
- 浏览器重新进入代码协作页面，Git 状态和文件差异正常加载。
- 付费 Provider 调用为 `0`。
