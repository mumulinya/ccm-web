# 飞书双通道命名与展示优化 v1

日期：2026-07-13  
状态：已完成

## 调整原因

设置页此前同时使用“飞书机器人”“通知机器人”“控制机器人”等名称，并且旧文案曾把任务状态描述为发送到固定 Webhook 群，容易让用户误解两条通道的职责。

## 统一命名

- `报告通知通道`：使用自定义机器人 Webhook，向固定飞书群发送日报、周报和手动报告通知。
- `任务会话通道`：使用飞书应用和 WebSocket 接收需求，并将计划、派发、TestAgent、验收和总结返回原任务会话。

## 页面调整

- 系统设置页标题统一为“飞书通道”，分别展示两个通道的用途、配置和健康状态。
- 报告通知区域不再声称发送任务状态。
- 任务会话区域明确说明进度会返回原会话。
- 自动开发页的飞书通知面板改名为“报告通知通道”，并说明任务进度不走固定报告群。

此次调整只优化用户可见命名和说明，不改变已经完成的底层路由、会话绑定和投递行为。
