# 清理中心工作台重构

## 目标

清理中心负责归档和永久删除任务、定时任务、项目运行、会话及执行证据。改版只调整信息架构和交互呈现，不改变后端清理定义、预览 Token、审计回执或数据边界。

## 页面结构

- 页头显示安全预览状态、最近扫描时间和刷新入口。
- 摘要条显示运行数据总量、已统计空间、可恢复整理数和永久操作数。
- 桌面端使用治理侧栏区分：
  - 存储概览：查看各类运行数据的数量和容量。
  - 安全整理：先归档，数据仍可恢复。
  - 永久删除：不可恢复的精确清除。
  - 清理记录：查看处理结果和审计信息。
- 移动端侧栏转换为顶部横向导航，四个视图保持可达。

## 存储概览

- 六类存储数据继续来自 `/api/cleanup/summary`，不建立第二套统计来源。
- 每项显示记录数、业务说明、占用空间、容量比例和进度条。
- 选择分类后展示最近 200 条明细；移动端明细表转换为纵向信息行。
- 明确提示清理中心不会处理项目源码、知识库和用户上传资料。

## 整理与删除

- 安全整理使用绿色可恢复语义，建议先归档再决定是否删除。
- 永久删除使用独立红色高风险区域。
- 生成预览不会修改数据。
- 桌面端预览清单固定在右侧检查栏，移动端按操作列表之后展示。
- 永久删除仍必须同时满足：
  - 使用当前有效的预览 Token。
  - 逐项选择要删除的记录。
  - 输入完全一致的“永久删除”确认短语。
- 预览明确显示过期时间和已选择数量，执行完成后进入清理记录。

## 验证

- `npm run build:frontend` 通过。
- `npm run test:cleanup-render` 通过 10 项页面、真实数据和门禁检查：
  - 桌面治理导航和六项容量分布。
  - 桌面存储概览无溢出。
  - 永久删除确认短语门禁。
  - 危险预览位于桌面右侧检查栏。
  - 清理记录默认折叠技术详情。
  - 移动端摘要和四个视图可达。
  - 移动端概览及危险预览无横向溢出。
  - 页面不展示原始 JSON 或预览 Token。
  - 正式 `/api/cleanup/summary` 能加载六类真实数据且页面无溢出。
- 桌面概览、危险预览、清理历史、移动概览和移动危险预览截图保存在 [`evidence`](./evidence/) 目录。
- `npm run test:cleanup` 的 4 项后端检查通过：隔离永久清除会删除关联执行产物、TestAgent 证据和回放并生成审计回执；正式接口的摘要、预览 Token 和显式确认门禁正常。
- 页面危险操作使用 mock 接口；永久清除仅在系统临时目录执行。正式接口只读取摘要并生成预览，不归档或删除真实数据，付费模型调用为 0。
