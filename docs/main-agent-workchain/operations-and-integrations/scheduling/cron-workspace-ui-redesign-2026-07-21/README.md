# 定时任务工作台 UI 重构

## 问题

旧页面使用八列表格同时展示名称、目标、Cron、提示词、状态、运行记录和全部操作。业务开发任务附带的可靠性信息较多，导致单行高度失控、操作分散，移动端则需要把每个表格单元机械堆叠，难以快速判断“哪个任务需要处理”。

## 新结构

- 页面顶部明确展示定时任务标题、刷新、归档和新建入口。
- 调度概览统一显示调度服务状态、全部、启用、停用、需要关注和正在执行数量。
- 搜索、状态和目标筛选集中在一条工具栏中；出现选择项后才展示批量操作条。
- 任务列表改为稳定的工作行，依次呈现：
  - 名称、目标、工作流和执行提示。
  - 时区、重试、需求认领、缺口续跑和通知策略。
  - 人类可读的执行频率与原始 Cron 表达式。
  - 最近状态、上次/下次时间、触发次数和结果摘要。
  - 启停开关、运行记录、立即运行、编辑和归档操作。
- 已知 Cron 表达式会显示为“每天 09:00”“工作日 09:00”“每 30 分钟”等可读计划，自定义表达式仍保留原文。

## 新建与编辑

- 弹窗采用固定标题、独立滚动正文和固定底部操作区。
- 字段按基础信息、调度可靠性、群聊工作流、执行计划和执行内容分区。
- 弹窗背景为不透明主题表面，不再透出后方任务列表。
- 桌面端可靠性参数使用两列布局；移动端自动切换单列底部面板。

## 兼容与修复

- 保留原有创建、编辑、启停、立即运行、批量操作、归档、恢复、永久清除和运行记录 API。
- 修复 `CronRunHistoryDrawer` 只在 composable 中导入、未注册到页面组件的问题；运行记录按钮现在会稳定打开抽屉。
- 移动端回归通过“更多 -> 定时任务”的真实导航进入页面。
- 移动端头部、概览和筛选区禁止被 Flex 压缩，长任务列表由页面纵向滚动承载。

## 验证

- `npm run build:frontend` 通过。
- `npm run test:cron-run-history-render` 通过真实本地 API 验证：
  - 搜索无结果与恢复结果。
  - 批量停用和重新启用。
  - 运行记录抽屉及折叠技术详情。
  - 新建任务可靠性表单。
  - `1440 x 920` 桌面和 `390 x 844` 移动端无横向溢出。
  - 移动端调度概览保持可见。
