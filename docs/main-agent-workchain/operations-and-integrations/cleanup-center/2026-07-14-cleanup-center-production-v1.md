# 清理中心生产化升级 v1

日期：2026-07-14

## 目标

将原有偏开发者工具的清理页面升级为可供普通用户安全使用的清理中心，并确保任务派发页与清理中心使用同一条永久清除链路。

## 用户界面

清理中心分为四个视图：

- 存储概览：任务、定时任务、项目运行、会话、执行产物、TestAgent 与回放证据。
- 安全整理：归档失败项目运行，归档后仍可复盘。
- 永久删除：按 7、30、90 天或全部归档范围生成候选清单。
- 清理记录：展示操作结果、处理数量、失败数量和释放空间。

页面不再直接展示原始 JSON、英文数据键或 ISO 时间。底层级联清理数量放入默认折叠的“技术详情”。

## 安全链路

永久删除必须依次经过：

1. 选择保留范围。
2. 生成有效期为 10 分钟的服务端预览快照。
3. 在候选清单中逐项勾选。
4. 输入“永久删除”。
5. 服务端校验预览令牌、所选 ID 和数据指纹。
6. 获取独占文件锁后执行。
7. 保存 `ccm-cleanup-receipt-v1` 清理收据。

预览后新增的归档记录不会被本次操作删除；预览中的记录状态发生变化时，服务端拒绝执行并要求重新预览。

## 完整任务清除

清理中心复用任务协作模块的 `purgeArchivedTask`，级联处理：

- 任务队列和运行状态；
- 子 Agent 会话与记忆快照；
- 执行记录、检查点和输出；
- 未清理的执行工作树；
- TestAgent 截图、浏览器证据和运行记录；
- 任务回放日志；
- 任务租约与取消状态。

## 数据边界

清理中心不会清理：

- 知识库与文档；
- 项目源码；
- 用户上传资料；
- 群聊记忆；
- 全局 Agent 长期记忆；
- 飞书消息队列与通道配置。

会话、测试证据和执行产物当前仅作为占用概览展示；只有在永久删除其归属任务时才会跟随完整链路清除。

## 审计存储

清理收据保存在：

`~/.cc-connect/cleanup-center/audit.jsonl`

单个日志超过 2 MB 时保留最近 500 条，页面默认读取最近 40 条。收据包含动作、保留范围、请求数量、成功与失败数量、释放空间、级联清理计数和完成时间。

## 自动化验证

新增命令：

- `npm run test:cleanup`
- `npm run test:cleanup-render`

隔离清理测试在临时用户目录创建任务、执行记录、检查点、输出、TestAgent 证据、TestAgent 运行记录和任务回放，再通过真实清理链路永久删除并逐项断言。

渲染回归覆盖：

- 桌面存储概览；
- 桌面永久删除预览；
- 桌面清理记录；
- 手机存储概览；
- 手机永久删除预览；
- 无横向页面溢出；
- 技术详情默认折叠；
- 未输入确认词时执行按钮禁用；
- 页面不泄露原始 JSON 或预览令牌。

截图与报告位于：
