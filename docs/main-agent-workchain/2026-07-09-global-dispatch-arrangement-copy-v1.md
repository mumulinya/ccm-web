# 全局主 Agent 派发安排文案收紧 v1

## 背景

全局主 Agent 的流式派发摘要里，旧兜底文案是“已完成安排，正在等待下游执行成员更新结果”。这句话描述的是安排动作完成，但用户容易误读成需求已经完成。

## 本次调整

- 前端 `GlobalAgent.vue` 的 `dispatch_launch_summary` 兜底改为“安排已发出，正在等待下游执行成员更新结果”。
- 静态自检 `preCompletionDispatchCopyAvoidsFalseDone` 增加前端断言：
  - 必须包含新的“安排已发出”文案。
  - 禁止回归“已完成安排”。

## 验收口径

派发、安排、进入任务链路都只代表开始执行或等待结果；只有任务卡验收、TestAgent/独立复核和最终总结通过后，才可对用户展示完成结果。
