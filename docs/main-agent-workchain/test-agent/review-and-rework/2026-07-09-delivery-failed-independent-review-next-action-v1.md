# Delivery Failed Independent Review Next Action v1

## 背景

CC 的交付要求是：如果验证或独立复核失败，必须如实说明失败，并在修复后重新验证，不能把失败状态包装成完成。此前交付报告在底层状态为 `done` 时，可能优先给出“可以查看改动详情”的完成式下一步，即使独立复核门禁没有通过。

## 本次修改

- 交付报告下一步优先识别复核、验证和验收缺口。
- 当独立复核未通过时，下一步明确显示：原执行成员按复核意见返工，修复后重新运行 TestAgent/独立复核，再给最终总结。
- 当验证失败或缺失时，下一步优先提示补齐验证证据。
- 只有没有阻塞性缺口时，完成状态才显示“可以查看改动详情”。

## 验证

- 运行时自测覆盖“状态为 done，但独立复核未通过”的场景。
- 静态自测覆盖返工下一步文案。
- 既有失败验证、计划缺口、普通完成和截图回归继续保留。
