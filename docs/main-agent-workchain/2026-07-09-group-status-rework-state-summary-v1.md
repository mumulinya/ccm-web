# Group Status Rework State Summary v1

## 背景

CC 风格的任务链路强调“复核失败 -> 原执行成员返工 -> 重新验收 -> 最终总结”。群聊主 Agent 的进展追问如果把 `reworking`、`needs_rework` 或阻塞状态泛化成“正在处理”，用户会看不出任务已经进入返工/待补齐阶段。

## 本次修改

- 群聊任务状态映射新增“返工中”和“待补齐”。
- `reworking`、`needs_rework`、`retrying`、`repairing` 会展示为“返工中”。
- `blocked`、`needs_attention`、`partial`、`missing_receipt`、`needs_info` 会展示为“待补齐”。
- 活跃任务选择会把返工/待补齐任务算作仍需跟进，避免被较新的已完成任务覆盖。
- 进展追问的下一步会明确说明：原执行成员按失败点或复核缺口返工，修复后重新验收和总结。

## 验证

- 运行时自测覆盖“较新的已完成任务 + 较早的返工任务”场景，确保仍展示返工任务。
- 静态自测覆盖“返工中”和“重新验收和总结”文案。
- 既有进展追问、用户待处理摘要、协议隐藏和截图回归继续保留。
