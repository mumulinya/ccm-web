# 交付报告验收结论 v1

## 背景

目标链路要求主 Agent 能形成清晰的计划、执行、验收和总结。此前任务卡已经能展示“主 Agent 验收”和“计划执行核对”，但统一交付报告的纯文本总结主要包含完成内容、涉及范围、验证结果、风险和下一步。

这会导致用户只看最终回复、全局历史或非前端消费方时，无法直接看到“这轮到底有没有经过主 Agent 验收”。参考 `D:\claude-code` 的用户可见输出原则，最终给用户的文本应当让用户不需要额外追问就能理解结果。

## 本次升级

- `ccm-main-agent-delivery-report-v1` 增加“验收结论”section。
- 交付报告增加 `acceptance` 字段，记录主 Agent 验收状态、门禁检查、验收标准和计划核对状态。
- Completion card 增加“验收”指标，和状态、范围、验证、风险一起展示。
- Pickup 摘要增加“验收：...”复看项，用户回来继续时能先看到是否已通过验收。
- 用户收尾建议的证据摘要增加验收结果。
- 前端终态兜底报告为失败/取消状态补“验收结论”。
- 任务卡本地总览在没有后端 completion card 时，也会显示验收指标和验收列表。

## 用户可见规则

- 完成态：显示“主 Agent 验收：已通过”或“已完成交付复核”。
- 失败态：显示“主 Agent 验收：未通过”，原因继续放在“未完成原因”里。
- 取消态：显示“主 Agent 验收：任务已停止，未继续验收”。
- 技术细节、原始回执、Trace 和执行器信息仍默认放在技术详情里。

## 验收点

- 统一交付报告 markdown 出现“验收结论”。
- Completion card 出现“验收”指标。
- Pickup 摘要出现验收复看项。
- 真实渲染截图中任务卡的交付报告和完成总览都能看到验收结论。
- 普通问话仍不展示 Todo 或任务卡。
