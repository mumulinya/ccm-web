# 用户收尾建议 v1

## 背景

对照 `D:\claude-code` 的 AgentTool 链路，子 Agent 的结果不会直接展示给用户，主 Agent 需要把执行结果消化成简短、可读、可操作的总结。项目里已经有计划、执行过程、验收、改动明细和交付总结，但“用户现在最该做什么”仍分散在多个 `next_action` 文案里。

## 本次升级

- 新增 `ccm-main-agent-user-handoff-v1`。
- 群聊主 Agent 任务卡在完成、失败、取消、撤销、阻塞、需要用户确认时输出 `user_handoff`。
- 全局主 Agent 历史卡、全局任务卡、项目执行卡通过统一前端归一化补齐同一结构。
- 前端 `TaskExperienceCard` 增加“接下来建议”区：展示一个主建议、最多三个辅助建议、证据摘要和未解决项。
- 技术内容只保留提示，底层记录、Trace、会话和执行器细节仍默认折叠在技术详情里。
- 普通问话不会生成任务卡，也不会显示收尾建议。

## 验证覆盖

- 后端 UX 自测检查完成任务展示收尾建议、普通问话不展示、用户可见文本不泄露内部协议词。
- 统一聊天自测检查全局完成、失败、取消三类历史卡都有正确收尾建议。
- Playwright 渲染回归检查群聊完成卡、全局完成卡、全局失败卡、全局取消卡的真实 DOM 展示。
