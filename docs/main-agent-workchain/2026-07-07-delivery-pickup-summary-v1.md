# 交付后回看摘要 v1

## 背景

参考 Claude Code 的 session / task summary 思路，用户离开一段时间后再回来，不应该只能从多条技术记录里重新拼接任务状态。CCM 已经有统一交付总结，但还缺一个更像“回来看这里”的短摘要，帮助用户快速知道任务完成了什么、有没有风险、下一步能做什么。

## 本次升级

- 统一交付报告 `ccm-main-agent-delivery-report-v1` 新增 `pickup_summary`。
- `pickup_summary` 包含当前状态、回看要点、下一步动作和“技术详情在哪里”的提示。
- 群聊主 Agent、全局主 Agent 只要使用统一交付报告，就能复用这份回看摘要。
- 普通问话仍不展示交付报告和回看摘要。
- Trace、原始回执、会话和执行记录继续默认收在“技术详情”。

## 用户可见效果

- 完成后的任务卡会显示“回来继续看这里”。
- 用户能快速看到改动、验证、风险和下一步，而不需要展开技术详情。
- 如果任务失败或取消，摘要会告诉用户如何恢复处理或重新发起。

## 验证

- `runMainAgentDeliveryReportSelfTest()` 覆盖 `pickup_summary` 结构和风险保留。
- 统一聊天任务体验自测验证全局历史完成报告会保留 `pickup_summary`。
- Playwright 渲染回归验证群聊任务卡和全局历史任务卡都能看到“回来继续看这里”。
