# 全局主 Agent 动作摘要待排查文案

日期：2026-07-09

## 背景

全局主 Agent 的流式“动作摘要”在动作失败或阻塞时，会显示“动作需要处理 / 需处理 1”。这类失败通常是主 Agent 要继续排查、重试或重新判断下一步，并不一定需要用户立即操作。

## 本次调整

- 将失败、错误、阻塞动作的状态标签从“需处理”改为“待排查”。
- 将动作摘要标题从“X 项动作需要处理”改为“X 项动作待排查”。
- 将最近动作文案从“某动作需要处理”改为“某动作待排查”。
- 新增全局流式失败动作 fixture，覆盖“执行遇到问题”的真实渲染路径。
- 新增 Playwright 回归和源码自测守卫，确保失败动作不会冒充用户待办。

## 验收点

- 成功动作仍显示“已返回”，不会提前宣称业务完成。
- 失败动作显示“待排查”，不会显示“需要处理/需处理”。
- 技术细节仍留在技术详情中，用户可见区只展示下一步判断需要。
