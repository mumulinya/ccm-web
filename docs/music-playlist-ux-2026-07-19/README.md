# 音乐页歌单操作体验优化

## 原有问题

- 新建歌单、选择目标歌单、播放全部和删除操作挤在同一条工具栏中。
- 单曲加入歌单依赖顶部全局下拉框，用户很难确认当前目标歌单。
- 歌单缺少统一管理入口，不能直接改名或调整歌曲顺序。
- 播放歌单后会强制跳到播放队列，打断当前浏览位置。
- 移动端工具栏空间不足，创建和管理入口不易操作。

## 本次调整

- 音乐库主视图显示全部、收藏、播放队列和歌单的实时数量。
- 增加独立“我的歌单”管理弹窗，支持新建、打开、播放、改名和删除。
- 每首歌固定提供“添加到歌单”入口，弹窗内直接选择目标歌单，也可以新建并立即添加。
- 已存在于目标歌单的歌曲显示“已添加”，避免重复操作。
- 当前歌单显示名称和曲目数，并提供播放全部、改名和删除操作。
- 歌单内支持歌曲上移、下移和移出；本地永久删除只在全部/收藏视图出现，降低误删风险。
- 播放当前歌单后保留歌单视图，不再强制跳转到播放队列。
- 后端阻止创建或改名为同名歌单。
- 移动端歌单管理使用底部适配弹窗，输入框和操作按钮独占可用宽度。

- `CCM_MUSIC_URL=http://127.0.0.1:3080/ npm run test:music-render`：通过。
- mock 流程覆盖创建、添加两首歌曲、排序、改名、桌面布局和移动端歌单弹窗。
- 真实页面验证歌单弹窗位于视口内，歌单打开、播放、改名和删除按钮均可见且不重叠。
- 测试过程未调用付费模型。
