# 知识库页面滚动修复

## 问题

知识库根节点使用 `min-height: 100%`，内容会把根节点直接撑高。外层标签页固定高度并使用 `overflow: hidden`，因此超出可视区域的内容被裁切，而知识库根节点自身没有形成可滚动溢出。

修复前实测：

- 知识库根节点 `clientHeight: 915px`
- 标签页父容器 `clientHeight: 646px`
- 根节点 `scrollHeight: 915px`
- 根节点无法产生有效滚动距离

## 修复

- 根节点改为 `height: 100%` 和 `min-height: 0`。
- 垂直方向使用独立滚动，水平方向隐藏溢出。
- 增加 `overscroll-behavior: contain` 和稳定滚动条槽位。
- 桌面和移动端共用同一滚动容器。

## 验证

- 前端 production build 通过。
- 桌面端根节点与父容器均为 `646px`，可滚动距离 `269px`，实测 `scrollTop: 268px`。
- 移动端根节点与父容器均为 `736px`，可滚动距离 `1018px`，实测到达底部。
- 桌面和移动端均可到达归档文档区域。
- 浏览器控制台 error 数为 0。
