# 全局页面加载遮罩

## 目标

首次打开懒加载页面时，不再展示无法区分加载与卡死的空白标签页。

## 实现

- App 中所有用户可见懒加载页面统一使用 `definePageComponent`。
- 页面资源开始加载后立即展示全尺寸加载遮罩。
- 遮罩提供加载状态、简短说明和持续进度反馈。
- 网络或 chunk 加载异常自动重试一次。
- 30 秒仍未完成时切换为明确错误页，并提供刷新按钮。
- 后台常驻的 `MusicRemoteHost` 不使用页面遮罩，避免覆盖当前页面。

## 覆盖范围

- 项目、群聊、全局 Agent
- 工具、任务、自动开发、终端、代码变更
- 知识库、记忆中心、清理中心、定时任务
- 设置、菜单、搜索、监控、宠物和音乐页面

## 验证

- frontend production build 通过。
- 受控延迟 `1.6s` 时加载遮罩数量为 1，存在“正在加载页面”状态。
- 页面完成加载后遮罩数量为 0，目标项目页面正常挂载。
- 模拟 chunk 连续失败后显示错误状态和“刷新页面”操作。
- `390 x 844` 移动端遮罩布局正常。
- 真实 `3080` 页面加载完成后无残留遮罩，浏览器控制台 error 数为 0。
