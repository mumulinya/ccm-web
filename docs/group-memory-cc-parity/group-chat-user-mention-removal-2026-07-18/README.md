# 群聊用户 @ 项目 Agent 入口移除

## 结果

- 群聊输入框不再提供 `@项目 Agent` 下拉菜单。
- 删除 @ 菜单的输入检测、键盘导航和插入逻辑。
- 输入提示不再宣传 @ 项目执行成员。
- 用户消息不再展示 `用户 -> @目标` 路由行。
- 项目分析和项目任务只切换消息模式，不改变消息接收者。

## 路由规则

- 群聊用户消息的前端目标固定为只读 `all`。
- `all` 在群聊服务端表示交给群聊主 Agent 编排。
- 项目子 Agent 的选择和任务分派继续由群聊主 Agent 完成。
- 后端内部 Agent 协作所需的项目标识和 Agent 间消息不受影响。

## 验证

- 前端 production build 通过。
- 输入 `@` 后 `.mention-dropdown` 数量为 0。
- 输入框提示不包含 `@`，用户消息路由标签数量为 0。
- 群聊渲染回归通过并生成 2 张截图。
- 浏览器控制台 error 数为 0。

现有大回归中的非本次失败：

- 统一会话任务体验自测仍有既有全局任务展示断言失败。
- 群聊业务链自测仍被既有 task Agent memory trusted-envelope 门禁阻止。
