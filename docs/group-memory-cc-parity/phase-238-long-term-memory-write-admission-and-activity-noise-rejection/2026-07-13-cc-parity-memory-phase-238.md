# CCM Memory CC Parity Phase 238

## 阶段目标

把群聊日志蒸馏从“发现候选就写入”升级为代码级长期记忆写入准入：

- 只保留跨会话仍有用、非显而易见、带原因或应用方式的稳定知识。
- PR 清单、Git 历史、周报/日报、最近改动、当前任务进度等活动流水不得进入长期记忆。
- 用户明确要求保存活动流水，也不能绕过长期价值门槛。
- 拒绝审计只保存候选标识、消息标识、类型和原因，不保存被拒绝正文。
- 旧通用蒸馏账本中的活动噪声会在下一次蒸馏时清退，并同步删除失去事实来源的生成文档。
- 每个 `groupId::gcs_*` 继续独立评估、写入和审计；Global Agent 不接收群聊会话正文。
- 旧群聊会话继续直接删除，不迁移。

## Claude Code 对照结论

Claude Code 在 `D:\claude-code\src\memdir\memoryTypes.ts:183-195` 把长期记忆排除项写成显式规则：不要保存代码模式和文件路径、Git 历史、调试方法、与 `CLAUDE.md` 重复的内容、一次性任务细节；即使用户明确要求保存 PR 或活动总结，排除规则仍然有效，只应保留真正令人意外且以后有用的部分。

CCM 之前的通用群聊日志蒸馏主要依赖消息角色和关键词分类。它可以生成类型化文档，但无法稳定区分“这次做了什么”和“以后为什么仍应这样做”，也没有对显式保存活动流水做硬拒绝。

Phase 238 将 Claude Code 的 write-side memory policy 落为确定性准入层。模型或用户可以提出候选，但是否进入长期记忆由平台按候选类型、跨会话价值、非显而易见程度、原因和应用方式重新判定。

## 实现内容

### 1. 长期写入准入契约

通用蒸馏候选可携带可选的 `memoryAdmission` 元数据：

```json
{
  "surprising": true,
  "nonObvious": true,
  "futureApplicable": true,
  "why": "为什么这个事实跨会话仍重要",
  "howToApply": "以后在什么条件下如何使用",
  "requestedByUser": false
}
```

这些字段只是候选证据，不是直接写入许可。平台仍会结合消息角色、候选类别和正文信号重新分类，并输出稳定的 admission reason。

### 2. 硬排除与分类门槛

以下通用候选直接拒绝：

- `completed_work`、`assignment`、`files`、`skills`、`verification`。
- PR/MR/提交清单、Git log/blame/history、活动/周报/日报、最近改动。
- 当前任务进度、临时状态、一次性交付细节。
- 没有跨会话价值的一次性完成、失败或阻塞。

可准入类别的门槛：

| 候选类型 | 最低条件 |
| --- | --- |
| 用户长期要求 | 存在跨会话持续适用信号 |
| 用户纠正 | 持久、非显而易见，并说明原因 |
| 失败/阻塞 | 可复现、持久、非显而易见，并说明原因 |
| 项目技术事实/派发决策 | 持久、非显而易见，并说明原因 |
| 外部资源 | 同时说明资源用途 |

用户纠正现在写入 `feedback/user_correction`，不再退化为普通用户要求。准入后的 Markdown 同时渲染 `Why` 和 `How to apply`，让后续子 Agent 得到可执行语义，而不只是事件摘要。

### 3. 旧噪声清退

每次通用蒸馏都会用同一准入规则复检旧 ledger facts：

- 不再满足门槛的旧事实从通用蒸馏账本中移除。
- 某一类别清空后，删除对应的陈旧生成 Markdown。
- 清退只作用于当前 `groupId::gcs_*` 的通用蒸馏制品，不读取或改写其他群聊会话。
- 生产三个活跃会话当前没有通用 session distillation ledger，因此本次没有生产数据需要清退。

### 4. 无正文拒绝审计

写入准入账本使用 `ccm-group-typed-memory-write-admission-v1`，记录：

- 本轮评估、准入、拒绝、硬排除和旧事实清退数量。
- candidate/message/category/type/reason 标识。
- 按类别和拒绝原因聚合的计数。
- 更新时间和版本。

拒绝观察不保存候选 text/content，也不会被渲染到子 Agent 上下文。账本最多保留 500 条观察，避免审计本身演变成新的活动日志。

### 5. 质量门与 Memory Center

类型化记忆蒸馏质量新增 fatal 检查 `long_term_write_admission`。如果产物包含未通过准入的事实，质量评估不能通过。

Memory Center 新增“长期记忆写入准入”诊断，展示：

- 原始候选、准入和拒绝数量。
- hard exclusion 数量。
- 清退旧噪声和删除陈旧文档数量。
- 无正文拒绝观察数量。
- 活动流水拒绝数量。

生产活跃会话当前没有通用蒸馏账本，因此面板按条件不渲染；Phase 238 隔离自测覆盖了完整有数据状态，没有向生产会话注入测试记录。
