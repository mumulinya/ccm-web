# Phase 118 - Ignore-Memory Receipt Repair Actions UI

## Goal

Continue the Claude Code parity memory goal by turning ignore-memory receipt diagnostics into actionable Memory Center repair operations.

Phase 117 made ignore-memory receipt compliance visible. This phase lets the group main Agent act on the generated repair work items directly from the Memory Center quality panel, including cross-group repair rows.

## Changed

- Added `ignoreMemoryReceiptWorkItemRows` to surface repair items from `worker_context_packet_ignore_memory_receipt_repair_work_items`.
- Added a dedicated repair work-item list under the ignore-memory receipt panel.
- Added actions for each visible work item:
  - claim,
  - dispatch marker,
  - complete,
  - block,
  - reopen.
- Generalized replay repair work item updates so they can operate on an explicit `groupId`, while keeping the existing selected-group detail behavior intact.
- Added responsive and dark-mode styling for the ignore-memory repair item list.
