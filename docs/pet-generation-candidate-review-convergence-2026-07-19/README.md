# 宠物候选图展示与收敛式重做

## 发现

任务 `petgen_mrrgyv2z_e96f07` 实际成功生成了三份 SVG 和三张渲染图，技术生成没有失败。三轮自动复审分数分别为 `56%`、`37%`、`57%`。

三轮均正确保留了蓝青开衫、白色内搭、左侧手机、右侧手链和中央吊坠，但持续出现圆脸、大头、短躯干和规则圆团盘发。第二轮分数下降说明旧重做策略没有收敛。

## 根因

- Skill 已写明禁止默认 Q 版，但旧重做只把自然语言 review 重新塞给模型。
- 下一轮仍然只看到原参考图，没有看到上一轮候选，因此每轮都是从头重画，无法保留已通过部分并定向修正。
- 参考图只显示到腰部，旧 review 却把推断出的完整头身比、裤腿和鞋型用于硬门禁，存在超出可见证据的要求。
- 三轮未达到自动阈值后被标记为技术失败，并隐藏候选图，用户无法判断、选择或继续修改。

## 新流程

- 生成记录展示每轮候选图片、分数、复审总结和最多三条具体偏差。
- 第二轮及后续生成同时附带原参考图与上一轮候选渲染图。
- 模型必须以候选为编辑基线，保留已通过的服装、配色、配件和左右关系，只修正失败项。
- Review 只对参考图中真实可见的身份特征做硬判断；不可见区域只检查设计连贯性。
- 三轮仍未过自动门禁时，选择最高分候选进入“等待用户决定”，不再作为技术失败。
- 用户可以明确选择任意候选，再继续修改或确认生成动作。
- 现有失败任务的候选文件会惰性投影到 API，不需要重新调用模型即可查看和选择。

## 数据与接口

- 候选 SVG：`preview-attempt-{n}.svg`
- 候选渲染：`preview-attempt-{n}.png`
- 候选设计设定：`preview-design-{n}.json`
- 候选复审：`preview-review-{n}.json`
- 候选图片：`GET /api/pets/generation-jobs/preview-attempt`
- 选择候选：`POST /api/pets/generation-jobs/adopt-attempt`
