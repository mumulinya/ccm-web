# 自动开发全宽工作台改造

## 问题

- 头部、标签栏和内容都限制在 `1280px`，宽屏主区域没有占满。
- 开发活动与阻塞信息上下堆叠，零数据时形成两个大空框。
- 状态、指标和操作缺少清晰的视觉识别。
- 移动端标题、操作和四项指标需要重新组织。

## 完成内容

- 删除 `1280px` 固定宽度，自动开发页面完整占用主内容区。
- 标题升级为“自动开发控制台”，补充 Agent Automation 语义层级。
- 刷新、立即运行、三个页面标签、链路状态和四项指标统一使用 Lucide 图标。
- 状态提示、指标条和内容区统一使用稳定的 8px 半径与主题变量。
- 宽屏下开发活动和阻塞与续跑并列展示，最近运行记录横跨主栏。
- 零活动状态提供“开始运行”操作；无阻塞状态说明未来哪些问题会显示在该区域。
- 右侧继续展示运行链路、无人值守和需求池，不改变原有数据与操作接口。
- 报告与通知视图同步解除宽度限制。
- 390px 移动端概览、工作复盘和通知设置均无横向溢出。

## 验证

- `npm run build:frontend`：通过。
- `node scripts/auto-dev-full-width-redesign-selftest.mjs`：通过。
- 宽屏内容宽度与主区域一致。
- 桌面端主栏双列和右侧链路栏检查通过。
- 移动端三个视图检查通过。
- 没有运行自动开发、生成报告或修改真实任务。
- 付费模型调用：`0`。
