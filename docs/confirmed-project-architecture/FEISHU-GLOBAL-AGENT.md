# 飞书全局 Agent 双向会话

Date: 2026-07-22

Status: Confirmed

## 会话模型

每个飞书来源会话映射到一个独立的全局 Agent 会话。飞书用户消息、全局 Agent 回复、运行状态与任务结果写入该精确会话；其他飞书群、其他用户会话和 Web 全局会话不会被混入。

## 入站流程

1. 飞书事件回调校验 Verification Token、事件类型、发送者和消息幂等 ID。
2. 根据 `chat_id + open_id` 解析或复用全局 Agent 会话。
3. 文字直接进入 Agentic Loop；图片和附件先下载、保存并走需求来源解析。
4. 全局 Agent 可以回答问题、查询系统、管理任务，或者把业务开发任务交给群聊主 Agent、项目 Agent 和 TestAgent 链路。
5. 新建的 run、mission 和 task 都追加到原飞书会话绑定。

## 出站流程

普通回复、等待确认、执行进度、运行中排队、停止结果、失败信息和最终总结，都优先通过会话绑定返回原 `chat_id/open_id`。如果原会话发送临时失败，消息进入持久 outbox，最多重试五次；只有完全不存在来源绑定时才降级到通用飞书报告通道。

## 权限流程

全局 Agent 下发的开发任务如果触发项目子 Agent 权限申请：

1. 权限申请记录 `originType=global` 和来源全局会话 ID。
2. 飞书通知包含项目、操作、风险、理由和申请 ID。
3. 用户可在原飞书会话回复 `批准权限 perm_xxx` 或 `拒绝权限 perm_xxx`。
4. 后端只接受属于当前精确全局会话且仍为 `awaiting_user` 的申请。
5. 批准后创建绑定任务、项目、子 Agent、Provider native session 和 checksum 的单次限时租约，并恢复原任务。
6. 兄弟飞书会话不能查询、批准或消费该权限。

任务派发中心和对应 Web 全局会话仍同步显示同一条权限申请，任何入口处理后其他入口都会刷新为已处理。

## 运行中消息

- `引导：内容`：在安全节点调整当前全局 Agent 运行。
- `排队：内容` 或普通新消息：当前任务结束后自动处理。
- `停止`：停止当前工作，保留已排队消息。
- 相同飞书消息 ID 不会重复执行。
