# 知识库使用系统业务流程确认

Date: 2026-07-22

Status: Confirmed

## 定位

知识库保存用户主动导入、在线同步或从会话手动归档的业务文档。它不是会话记忆，也不会把所有聊天自动写成文档。

- 会话记忆负责当前单个会话的连续性与压缩。
- 长期记忆负责跨会话仍然有效的决定、约束和纠正。
- 知识库负责可核验的规范、说明书、需求文档、接口资料和其他文件原文。

知识库命中后以 `[source:文件名#分片序号]` 作为引用。知识文档属于不可信资料层，不能覆盖系统指令、权限或用户当前要求。

## 会话使用范围

| 使用入口 | 自动使用 | 可继续深读 | 允许范围 |
| --- | --- | --- | --- |
| 全局 Agent 当前会话 | 是，每轮按用户问题召回 | `query_knowledge` | 全局知识；受限全局知识只允许全局 Agent |
| 群聊主 Agent 当前 `gcs_*` 会话 | 是，每轮召回 | 主 Agent 的知识工具 | 当前群聊、共享全局、群成员项目的共享知识 |
| 独立项目 Agent 当前项目会话 | 是，普通问答和开发任务均召回 | 签名 `ccm__knowledge_context` MCP | 当前项目、共享全局；不读取其他项目或群聊 |
| 群聊项目子 Agent 当前 `tas_*` 任务 | 是，由主 Agent 工作单携带命中结论 | 签名 MCP 的 `search_knowledge`、`read_knowledge_document` | 当前群聊、目标项目、共享全局 |
| TestAgent | 按绑定工作单和签名作用域 | 签名 MCP | 只读当前验收任务允许的范围 |
| 音乐 Agent | 否 | 否 | 音乐 Agent 使用曲库、播放历史和音乐偏好，不读取开发知识库 |

群聊主 Agent 不会读取兄弟群聊。独立项目 Agent 不会读取兄弟项目。群聊主 Agent 可以读取成员项目标为 `shared` 的资料，但不会仅因项目属于该群就读取项目的 `restricted` 文档。

## 每轮检索流程

1. 根据当前 Agent 的精确角色、群聊 ID、项目名和任务绑定计算允许访问的文件清单。
2. 权限过滤在相关性打分之前执行，无权文档不会参与候选排名。
3. 有可用 Embedding 配置时，使用远程语义向量并结合本地关键词、中文切词、覆盖率进行混合排序。
4. 没有配置 Embedding 时，直接使用本地关键词、中文切词和 hashing 向量混合检索，不发起外部请求。
5. 远程 Embedding 调用失败时自动切换本地检索，并保留真实的 fallback 状态与错误信息。
6. 自动召回只注入相关分片和引用，不把整个知识库塞入上下文。
7. 第三方 Agent 需要更多原文时通过签名 MCP 分页读取；模型不能传入其他 scope 绕过绑定。

## 文档进入与更新

- 用户可在“知识库与文档”页面上传文件、导入在线地址、配置同步目录或从会话手动保存知识。
- 导入时必须选择 `global`、`group`、`project` 或 `agent` 作用域，并可设置 `shared` 或 `restricted`。
- 文档更新保留版本与来源；删除文档后不再参与新检索。
- 定时任务、任务派发的附件默认属于该任务的输入，不会自动变成知识库文档。需要长期复用时由用户或验收流程明确归档。

## Embedding 配置

Embedding 是语义匹配增强项，不是知识库可用性的前置条件。

- 未配置：页面显示“本地混合检索（无需配置）”。
- 已配置：远程语义检索与本地排序共同工作。
- 配置失效：当前请求自动本地回退，不中断 Agent；技术详情显示回退与错误。

任何模式下，原始文档、作用域权限和引用标识保持一致。
